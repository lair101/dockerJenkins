Tasks to complete after generating the Healthcare: Web service to DICOM pattern

Tasks that you must complete after you have generated the pattern.

IBM Integration Bus Deployment

  1. Create the BAR file and add your message flow to it:
    1. Click File > New > BAR file.
    2. Enter the name of your application or select one from the displayed list. The list is filtered to show only projects in the active working set.
    3. Enter a name for the BAR file that you are creating.
    4. Click Finish. A file with a .bar extension is created and is displayed in the Application Development view, under the BARs folder. The BAR file editor opens.
    5. On the Prepare tab, select the generated application to add it to your BAR file.
    6. Select Compile and in-line resources and then click Build and Save to create your BAR file.
  2. Configure the BAR file. Further configuration of the message flow in the broker archive file is not required before you deploy the message flows for your Healthcare: Web service to DICOM pattern instance, but you might want to review some of the default options and configure changes for your environment. In the BAR file editor, on the Manage tab:
    1. Optional: Use the LoggingOn user-defined property to select whether logging is on or off for the web service message flow.
    2. Optional: Use the ErrorLoggingOn user-defined property to select whether error messages are generated.
    3. Optional: You can change any of the configurable properties in the BAR file, for example, properties for security or creating additional instances.

      You must ensure that any values that you change do not affect the behavior of the pattern instance.

  3. Create queues. WebSphere MQ queues must be created if they do not already exist for the error and log messages.
    1. Create WebSphere MQ queues. You can create WebSphere MQ resources either on the command line by using programs and utilities such as runmqsc, or by using the WebSphere MQ graphical interface. For more information about creating WebSphere MQ resources, see the WebSphere MQ product documentation, available on the WebSphere MQ library web page.
  4. Deploy the BAR file to an integration server.

After you have created a BAR file that contains your pattern, you can test it by using the DICOM test application. You can use the DICOM test application to send and receive DICOM messages, and to send and receive web service messages. For more information about using the DICOM test application, see "Using the DICOM test application" in the IBM Integration Bus Healthcare Pack documentation.
