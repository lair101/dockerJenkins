Constraints on the use of the Healthcare: Web service to DICOM pattern

A number of constraints apply when you use the Healthcare: Web service to DICOM pattern.

You can use the Healthcare: Web service to DICOM pattern only if you meet all of the following constraints:

  • Requests must be received as SOAP messages over HTTP.
  • Requesting applications must support a synchronous request response interaction style.
  • Requesting applications must support the XML format for search criteria and results defined by the Healthcare: Web service to DICOM pattern.
  • The DICOM PACS must support C-MOVE commands if the DICOM instances are moved to a destination application.
  • The DICOM PACS must support either the patient or study query root for C-FIND commands.
  • If logging or error handling is required, the format used by the Healthcare: Web service to DICOM pattern is assumed to meet your requirements.
