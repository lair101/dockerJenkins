Computer connection

The driver supports both LAN and serial connections.

Please note that for the Eagle and Tramscope devices, the serial ports are RS422, not RS232. In order to connect a standard PC serial port, an RS232 to RS422 converter is required (the advantage of RS422 is that the cable can be 1200m long).

Serial (Tramscope and Eagle - RS422)

There are two possible serial connections, both on the patient monitor itself and on the Tram-net hub. In both cases the ports are labelled ASYNC COMM. The same patient data is available at either connection point.

Cable wiring for ASYNC COMM ports - refer to documentation of your RS232 to RS422 converter for wiring configuration.

LAN (GE Unity)

Each device is assigned an IP address on the Unity network. This address is used as an identifier in the CMSense configuration message. The driver will use UDP port 7000 to listen for devices on the network, and then communicate with the device that has the matching IP address.

A minimal configuration message should look like;

<CMLLC type="CMLLCSockets">
<CMLLCCfg identifier="X.X.X.X" ipaddress="" type="UDP" port="7000" />
</CMLLC>

where X.X.X.X is the IP address of the device.

Supported communication parameters:

Baud rate 9600
Parity None
Data bits 8
Stop bits 1
Flow control None
