Known issues:

  • If the driver is not stopped or closed down gracefully, the monitor will continue to try and send messages to a client that is no longer listening. It has been found that after a short while the monitor no longer responds to communication, and requires the power to be cycled before communication can be reestablished (the duration is dependent on the amount of data being sent. In the case of waveforms it may be less than a minute). It is always recommended that drivers should have the stop() method called before an application is closed down, and in this case, to not do so may mean the monitor will not send any data when reconnected without powering the device down and restarting it first.

Supported Data Parameters

The following waveforms are supported. For all waveforms, the primaryid is 111:889::, with the subid being a value from the following table. To start a waveform make a request with the value CM_START. To stop waveform transmission, send the same request with the value CM_STOP (see SDK manual for full details).

The Datex Ohmeda can support a maximum of 600 samples per second, which can be made from any number of the following waveforms. Requests for waveforms once this limit has been reached will be ignored. For example, 1 ECG channel and 3 Invasive blood pressure channels will fulfill this maximum requirement, and no further waveforms can be requested.
