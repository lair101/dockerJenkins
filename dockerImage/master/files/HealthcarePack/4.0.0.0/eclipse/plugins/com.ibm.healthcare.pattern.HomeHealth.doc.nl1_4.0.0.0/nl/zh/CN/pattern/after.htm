生成“Healthcare：Home Health”模式后要完成的任务

在生成“Healthcare：Home Health”模式的实例后，请完成以下步骤：

  • 在“Healthcare：Home Health”模式中实施用户定义的子流：
    • “Healthcare：Home Health”模式使用 UserDefinedFunction.subflow 来处理您的 HL7 V2.6 消息。如果想要定制模式处理这些消息的方式，请转至 Subflows/UserDefinedFunction.subflow。

      在入局消息处理完成后，会将一个应答返回给调用应用程序或设备。

  • 可选：如果期望“Healthcare：Home Health”模式接收大量请求，请配置系统以使用相应的 HTTP 拓扑。请参阅 IBM Integration Bus 文档中的配置 HTTP 拓扑。
  • 可选：如果配置“Healthcare：Home Health”模式启用 SAML 2.0 令牌通过，请完成以下步骤：
    1. 修改并运行 SecurityProfile scripts 文件夹中提供的示例可配置服务脚本以定义外部 SAML 安全令牌服务的安全概要文件。 有关更多信息，请参阅 IBM Integration Bus 文档中的用于认证和授权的 SAML 令牌功能。
    2. 创建包含 Healthcare：Home Health 应用程序的 BAR 文件。
    3. 在 BAR 文件编辑器中，使用您在步骤 1 中定义的安全概要文件配置 SecurityPEP 节点。有关更多信息，请参阅 IBM Integration Bus 文档中的配置身份或安全令牌的抽取。
