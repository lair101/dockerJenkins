Tareas que deben completarse después de generar el patrón Servicio sanitario: Salud en el hogar

Después de generar una instancia del patrón Servicio sanitario: Salud en el hogar, siga los pasos siguientes:

  • Implemente el subflujo definido por el usuario en el patrón Servicio sanitario: Salud en el hogar
    • El patrón Servicio sanitario: Salud en el hogar utiliza UserDefinedFunction.subflow para procesar sus mensajes HL7 v2.6. Si desea personalizar la forma en que el patrón procesa estos mensajes, vaya a Subflows/UserDefinedFunction.subflow.

      Una vez completado el proceso del mensaje entrante, se devuelve un acuse de recibo a la aplicación o dispositivo de llamada.

  • Opcional: Si se espera que el patrón Servicio sanitario: Salud en el hogar reciba un número elevado de solicitudes, configure el sistema para que utilice una topología HTTP adecuada. Consulte Configuración de la topología HTTP en la documentación de IBM Integration Bus.
  • Opcional: Si configura el patrón Servicio sanitario: Salud en el hogar para habilitar el paso a través de señal SAML 2.0, siga los pasos siguientes:
    1. Modifique y ejecute los scripts de servicio configurable que se proporcionan en la carpeta SecurityProfile scripts para definir un perfil de seguridad para un Servicio SAML de token seguro. Para obtener más información, consulte Posibilidades de autenticación y autorización de las señales SAML en la documentación de IBM Integration Bus.
    2. Cree un archivo BAR que contenga la aplicación Servicio sanitario: Salud en el hogar.
    3. En el editor de archivos BAR, configure el nodo SecurityPEP con el perfil de seguridad que ha definido en el paso 1. Para obtener más información, consulte Configuración de la extracción de una señal de identidad o de seguridad en la documentación de IBM Integration Bus.
