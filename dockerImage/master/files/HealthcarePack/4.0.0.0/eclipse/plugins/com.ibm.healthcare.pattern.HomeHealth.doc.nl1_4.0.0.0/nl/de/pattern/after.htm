Tasks, die nach der Generierung des Musters 'Healthcare: Ambulante Pflege' auszuführen sind

Führen Sie folgende Schritte aus, nachdem Sie eine Instanz des Musters 'Healthcare: Ambulante Pflege' generiert haben:

  • Implementieren Sie den benutzerdefinierten untergeordneten Nachrichtenfluss im Muster 'Healthcare: Ambulante Pflege':
    • Das Muster 'Healthcare: Ambulante Pflege' verwendet UserDefinedFunction.subflow zur Verarbeitung Ihrer HL7-Nachrichten der Version 2.6. Wenn Sie die Verarbeitung dieser Nachrichten durch das Muster anpassen möchten, wechseln Sie zu 'Subflows/UserDefinedFunction.subflow'.

      Nachdem die eingehende Nachricht verarbeitet wurde, wird eine Bestätigung an die aufrufende Anwendung bzw. das aufrufende Gerät gesendet.

  • Optional: Konfigurieren Sie Ihr System für die Verwendung einer entsprechenden HTTP-Topologie, wenn für das Muster 'Healthcare: Ambulante Pflege' eine große Anzahl von Anforderungen erwartet wird. Weitere Informationen finden Sie unter Configuring your HTTP topology (HTTP-Topologie konfigurieren) in der Dokumentation zu IBM Integration Bus.
  • Optional: Führen Sie folgende Schritte aus, wenn Sie das Muster 'Healthcare: Ambulante Pflege' für die Aktivierung des SAML 2.0-Tokendurchgriffs konfigurieren möchten:
    1. Ändern Sie die im Ordner SecurityProfile scripts (Sicherheitsprofilscripts) bereitgestellten Beispiele für konfigurierbare Service-Scripts und führen Sie diese aus, um ein Sicherheitsprofil für einen externen sicheren SAML-Token-Service zu definieren. Sie finden weitere Informationen unter SAML token capabilities for authentication and authorization (SAML-Tokenfunktionen für die Authentifizierung und Autorisierung) in der Dokumentation zu IBM Integration Bus.
    2. Erstellen Sie eine BAR-Datei, die die Anwendung "Healthcare: Ambulante Pflege" enthält.
    3. Konfigurieren Sie im Editor für BAR-Dateien den SecurityPEP-Knoten mit dem in Schritt 1 definierten Sicherheitsprofil. Weitere Informationen finden Sie im Abschnitt Extraktion einer Identität oder eines Sicherheitstokens konfigurieren in der Dokumentation zu IBM Integration Bus.
