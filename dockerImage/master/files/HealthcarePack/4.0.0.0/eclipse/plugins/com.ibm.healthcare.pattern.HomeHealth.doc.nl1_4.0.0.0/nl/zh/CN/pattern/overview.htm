“Healthcare：Home Health”模式

使用“Healthcare：Home Health”模式从移动或 PC 设备接受 Web 服务请求以从 Home Health 设备发送数据。

上下文

患者持有 Home Health 设备（例如，温标或心脏监护仪），他们可用于测量自己的健康状况。这些设备连接到用于转发此信息的应用程序托管的设备，例如移动或家用 PC。医疗专业人员需要知道 AHD 的读数，这样他们可以监视患者的健康状况。要准确监视患者的健康状况，需要定期读取数据。

目标

医疗专业人员需要通过广域网 (WAN) 从 Home Health 设备接收数据，并在请求应用程序中查看数据。他们需要一个系统，用于将请求应用程序与用来从 Home Health 设备接收信息的 AHD 集成。此系统必须能够接收 HL7 消息类型未经请求的观察结果 (ORU_R01) 的消息，并且返回消息已接收的确认。

解决方案

“Healthcare：Home Health”模式是提供以下功能的开发加速器：

  • 抽取并验证来自接收到的 SOAP 请求的 HL7 消息类型未经请求的观察结果 (ORU_R01) 的流和子流。
  • 用户定义的接收到的消息处理。
  • 将相应的确认发送回客户机。
  • 您可用于通过 SAML 设置安全的可配置服务脚本。
