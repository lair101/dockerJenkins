Patrón Servicio sanitario: Salud en el hogar

Utilice el patrón Servicio sanitario: Salud en el hogar para aceptar solicitudes de servicio web procedentes de dispositivos móviles o de ordenadores personales que envíen datos desde dispositivos de Salud en el hogar.

Contexto

Los pacientes poseen dispositivos de Salud en el hogar, por ejemplo, básculas o monitores cardíacos, que puede utilizar para medir su salud. Estos dispositivos se conectan a un dispositivo de alojamiento de aplicaciones (AHD), por ejemplo un móvil o un ordenador personal, que reenvía esta información. Un profesional médico tiene que conocer las lecturas de los AHD, para poder supervisar la salud de sus pacientes. Para poder supervisar la salud de los pacientes, es necesario realizar lecturas regulares.

Objetivo

Un profesional médico quiere recibir los datos procedentes de dispositivos Salud en el hogar en una WAN (Wide Area Network) y visualizar los datos en su aplicación solicitante. Necesitan un sistema que integre su aplicación solicitante con un AHD que reciba información del dispositivo de Salud en el hogar. Este sistema debe poder recibir mensajes del tipo de mensaje HL7 Unsolicited Observation Results (ORU_R01) y devolver acuses de recibo confirmando que se reciben los mensajes.

Solución

El patrón Servicio sanitario: Salud en el hogar es un acelerador del desarrollo que proporciona las siguientes funciones:

  • Flujos y subflujos que extraen y verifican el tipo de mensaje HL7 Unsolicited Observation Results (ORU_R01) de las solicitudes SOPA recibidas.
  • Procesamiento definido por el usuario del mensaje recibido.
  • Devolución del acuse de recibo adecuado al cliente.
  • Scripts de servicio configurable que puede utilizar para configurar la seguridad, utilizando SAML.
