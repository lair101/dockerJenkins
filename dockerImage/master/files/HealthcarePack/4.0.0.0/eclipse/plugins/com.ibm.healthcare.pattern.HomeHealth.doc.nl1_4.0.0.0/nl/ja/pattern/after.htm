「Healthcare: Home Health」パターンを生成した後に完了するタスク

「Healthcare: Home Health」パターンのインスタンスを生成した後に、以下の手順を完了します。

  • 「Healthcare: Home Health」パターンにユーザー定義のサブフローを実装します。
    • 「Healthcare: Home Health」パターンでは、UserDefinedFunction.subflow を使用して HL7 v2.6 メッセージを処理します。パターンがメッセージを処理する方法をカスタマイズする場合は、Subflows/UserDefinedFunction.subflow に移動してください。

      着信メッセージの処理が完了すると、肯定応答が呼び出しアプリケーションまたはデバイスに返されます。

  • オプション: 「Healthcare: Home Health」パターンで多数の要求を受け取ることが予想される場合は、適切な HTTP トポロジーを使用するようにシステムを構成します。IBM Integration Bus の資料にある『HTTP トポロジーの構成』をご覧ください。
  • オプション: 「Healthcare: Home Health」パターンの構成で SAML 2.0 トークン・パススルーを有効にする場合は、以下の手順を実行します。
    1. SecurityProfile scripts フォルダーに用意されているサンプル構成可能サービス・スクリプトを変更して実行し、外部 SAML セキュア・トークン・サービスのセキュリティー・プロファイルを定義します。詳細については、IBM Integration Bus の資料にある『認証および許可用の SAML トークンの機能』をご覧ください。
    2. 「Healthcare: ホーム・ヘルス」アプリケーションを組み込んだ BAR ファイルを作成します。
    3. BAR ファイル・エディターを開き、手順 1 で定義したセキュリティー・プロファイルを使用して SecurityPEP ノードを構成します。 詳細については、IBM Integration Bus の資料にある『ID またはセキュリティー・トークンの抽出の構成』をご覧ください。
