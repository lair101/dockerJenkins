「Healthcare: Web service to DICOM」パターンの使用に関する制約

「Healthcare: Web service to DICOM」パターンを使用する場合に適用される制約は多数あります。

以下のすべての制約を満たす場合にのみ、「Healthcare: Web service to DICOM」パターンを使用できます。

  • HTTP を介した SOAP メッセージとして要求を受け取る必要がある。
  • 要求元アプリケーションが同期要求応答対話スタイルをサポートしている必要がある。
  • 要求元アプリケーションが「Healthcare: Web service to DICOM」パターンによって定義される検索基準および結果の XML 形式をサポートしている必要がある。
  • DICOM インスタンスが宛先アプリケーションに移動される場合、DICOM PACS が C-MOVE コマンドをサポートしている必要がある。
  • DICOM PACS が C-FIND コマンドに対して患者か研究照会ルートのいずれかをサポートしている必要がある。
  • ロギング処理かエラー処理が必要な場合、「Healthcare: Web service to DICOM」パターンで使用されるフォーマットが要件を満たしていると想定される。
