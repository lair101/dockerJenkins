Restricciones de uso del patrón Servicio sanitario: Servicio web a DICOM

Cuando se utiliza el patrón Servicio sanitario: Servicio web a DICOM, se aplican una serie de restricciones.

Solo puede utilizar el patrón Servicio sanitario: Servicio web a DICOM si cumple todas las restricciones siguientes:

  • Las solicitudes deben recibirse como mensajes SOAP a través de HTTP.
  • Las aplicaciones solicitantes deben dar soporte a un estilo de interacción de solicitud y respuesta síncronas.
  • Las aplicaciones solicitantes deben dar soporte al formato XML para los criterios de búsqueda y resultados definidos por el patrón Servicio sanitario: Servicio web a DICOM.
  • El PACS DICOM debe dar soporte a los mandatos C-MOVE si las instancias DICOM se trasladan a una aplicación de destino.
  • El PACS DICOM debe dar soporte a la raíz de consulta de paciente o de estudio para los mandatos C-FIND.
  • Si se necesita manejo de errores o registro, se presupone que el formato utilizado por el patrón Servicio sanitario: Servicio web a DICOM satisface sus requisitos.
