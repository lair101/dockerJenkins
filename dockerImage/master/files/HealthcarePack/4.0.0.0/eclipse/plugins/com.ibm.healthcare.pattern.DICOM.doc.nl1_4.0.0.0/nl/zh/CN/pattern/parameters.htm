“Web 服务到 DICOM”模式的参数

DICOM 服务
模式参数 缺省值 描述
应用程序实体标题 BROKERAE

这是模式生成的应用程序实体在连接到远程应用程序实体 (PACS) 时使用的 DICOM 标题。
远程应用程序实体标题 PACS

这是 DICOM C-FIND 和 C-MOVE 命令所发送到的远程应用程序实体的 DICOM 标题。
最大结果数 99

此参数指定处理的 C-FIND 响应消息的最大数目。此值确保长时间运行的 C-FIND 查询在合理的时间长度内完成。
连接超时（秒） 60

这是允许与远程应用程序实体 (PACS) 建立连接的最大时间长度。
连接详细信息 localhost:11112

此参数以 hostname:port 格式定义远程应用程序实体 (PACS) 的 TCP/IP 连接。
启用针对查询 WSDL 的支持 选中 如果选中此参数，并且集成节点接收到包含 ?wsdl 查询字符串的 HTTP GET 请求，那么集成节点将返回与端点相关的 WSDL 和 XML 模式信息。使用此参数可以控制 WSDL 的分发。
日志记录
模式参数 缺省值 描述
需要进行日志记录 取消选中 此模式参数确定模式实例是否包含用于进行日志记录的子流。

如果未选中此模式参数，那么将不包含日志记录子流。

如果选中此模式参数，那么必须为“日志队列管理器”和“日志队列”设置值。
日志队列管理器 此模式参数定义用于进行日志记录的队列管理器。如果使用了集成节点队列管理器进行日志记录，那么可以保留此参数为空。

仅当选中了“需要进行日志记录”时，才需要指定此模式参数。
日志队列 日志 此模式参数定义用于进行日志记录的队列。

仅当选中了“需要进行日志记录”时，才需要指定此模式参数。
错误处理
模式参数 缺省值 描述
需要错误消息 选中 此模式参数确定模式实例是否包含用于将错误消息放入队列的代码。

如果选中了此模式参数，那么必须为“错误队列管理器”和“错误队列”设置值。
错误队列管理器 此模式参数定义用于进行错误日志记录的队列管理器。

仅当选中了“需要错误消息”时，才需要指定此模式参数。
错误队列 ERROR 此模式参数定义用于存放错误消息的队列。

仅当选中了“需要错误消息”时，才需要指定此模式参数。
常规
模式参数 缺省值 描述
流前缀 此模式参数定义要在所有消息流名称的开头添加的前缀。
流后缀 此模式参数定义要在所有消息流名称的末尾添加的后缀。
队列前缀 此模式参数定义要在所有队列名称的开头添加的前缀。

如果使用了缺省队列名称，那么必须设置前缀和/或后缀，以标识每个模式实例的队列名。
队列后缀 此模式参数定义要在所有队列名称的末尾添加的后缀。

如果使用了缺省队列名称，那么必须设置前缀和/或后缀，以标识每个模式实例的队列名。
简短描述 此模式参数定义要对生成的所有消息流的“简短描述”属性添加的描述。

所有模式参数均随其缺省值一起列出。 未指示缺省值时，表明缺省值是空字符串。
