生成“Healthcare：Web 服务到 DICOM”模式后需要完成的任务

生成此模式后必须完成的任务。

IBM Integration Bus 部署

  1. 创建 BAR 文件并向其添加消息流：
    1. 单击“文件”>“新建”>“BAR 文件”。
    2. 输入应用程序的名称，或者从显示的列表中选择名称。此列表已进行过滤，仅显示活动工作集中的项目。
    3. 为您要创建的 BAR 文件输入名称。
    4. 单击“完成”。这将创建一个扩展名为 .bar 的文件，该文件将显示在“应用程序开发”视图中的 BARs 文件夹下。BAR 文件编辑器将打开。
    5. 在“准备”选项卡上，选择生成的应用程序，以便将其添加到 BAR 文件中。
    6. 选择“编译并内联资源”，然后单击“构建并保存”以创建 BAR 文件。
  2. 配置 BAR 文件。在部署“Healthcare：Web 服务到 DICOM”模式实例的消息流之前，不需要在代理归档文件中对消息流进行进一步配置，但您可能希望复查某些缺省选项并针对您的环境配置更改。 在 BAR 文件编辑器中的“管理”选项卡上：
    1. 可选：使用用户定义属性 LoggingOn 来选择要对 Web 服务消息流开启还是关闭日志记录。
    2. 可选：使用用户定义属性 ErrorLoggingOn 来选择是否生成错误消息。
    3. 可选：可以在 BAR 文件中更改任何可配置属性，例如，安全性属性或者用于创建其他实例的属性。

      您必须确保更改的任何值都不会影响模式实例的行为。

  3. 创建队列。如果尚未存在用于存放错误消息和日志消息的 WebSphere MQ 队列，那么必须进行创建。
    1. 创建 WebSphere MQ 队列。可以从命令行使用 runmqsc 之类的程序和实用程序来创建 WebSphere MQ 资源，也可以使用 WebSphere MQ 图形界面完成此任务。有关创建 WebSphere MQ 资源的更多信息，请参阅 WebSphere MQ 资料库 Web 页面上的 WebSphere MQ 产品文档。
  4. 将 BAR 文件部署到集成服务器。

在创建包含模式的 BAR 文件之后，可使用 DICOM 测试应用程序对其进行测试。可使用 DICOM 测试应用程序来发送和接收 DICOM 消息，以及发送和接收 Web 服务消息。有关使用 DICOM 测试应用程序的更多信息，请参阅 IBM Integration Bus Healthcare Pack 文档中的“使用 DICOM 测试应用程序”。
