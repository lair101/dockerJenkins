Tareas que deben completarse después de generar el patrón Servicio sanitario: Servicio web a DICOM

Tareas que deben completarse después de generar el patrón.

Despliegue de IBM Integration Bus

  1. Cree el archivo BAR y añádale el flujo de mensajes:
    1. Pulse Archivo > Nuevo > Archivo BAR.
    2. Escriba el nombre de su aplicación o seleccione uno de lista visualizada. La lista se filtra para mostrar sólo los proyectos del conjunto de trabajo activo.
    3. Especifique un nombre para el archivo BAR que está creando.
    4. Pulse Finalizar. Se crea un archivo con la extensión .bar y se muestra en la vista Desarrollo de aplicaciones, bajo la carpeta BARs. Se abre el Editor de archivos BAR.
    5. En el separador Preparar, seleccione la aplicación generada para añadirla al archivo BAR.
    6. Seleccione Compilar e incorporar recursos y, a continuación, pulse Compilar y guardar para crear el archivo BAR.
  2. Configure el archivo BAR. No se requiere más configuración del flujo de mensajes en el archivo de archivador de intermediario antes de desplegar los flujos de mensajes para la instancia de patrón Servicio sanitario: Servicio web a DICOM, pero quizás desee revisar algunas de las opciones predeterminadas y configurar cambios en su entorno. En el editor de archivos BAR, en el separador Gestionar:
    1. Opcional: utilice la propiedad definida por el usuario LoggingOn para seleccionar si el registro está activado o desactivado para el flujo de mensajes de servicio web.
    2. Opcional: utilice la propiedad definida por el usuario ErrorLoggingOn para seleccionar si se generan mensajes de error.
    3. Opcional: puede cambiar cualquiera de las propiedades configurables en el archivo BAR, por ejemplo, las propiedades de seguridad o crear instancias adicionales.

      Debe asegurarse de que los valores que cambie no afectarán al comportamiento de la instancia de patrón.

  3. Cree colas. Se deben crear colas para WebSphere MQ, si no existen ya, para los mensajes de error y de registro.
    1. Cree colas de WebSphere MQ. Puede crear recursos de WebSphere MQ en la línea de mandatos utilizando programas y utilidades como por ejemplo runmqsc, o utilizando la interfaz gráfica de WebSphere MQ. Para obtener más información sobre cómo crear recursos para WebSphere MQ, consulte la documentación de WebSphere MQ, disponible en la página web de la biblioteca de WebSphere MQ.
  4. Despliegue el archivo BAR en un servidor de integración.

Una vez que haya creado un archivo BAR que contenga el patrón, podrá probarlo mediante la aplicación de prueba de DICOM. Puede utilizar la aplicación de prueba de DICOM para enviar y recibir mensajes de DICOM y para enviar y recibir mensajes de servicio web. Para obtener más información sobre cómo utilizar la aplicación de prueba de DICOM, consulte "Utilización de la aplicación de prueba de DICOM" en la documentación de IBM Integration Bus Healthcare Pack.
