Tasks, die nach der Erstellung des Musters Healthcare: Web-Service an DICOM auszuführen sind

Tasks, die Sie nach der Erstellung des Musters ausführen müssen.

IBM Integration Bus implementieren

  1. Erstellen Sie die BAR-Datei und fügen Sie ihr Ihren Nachrichtenfluss hinzu:
    1. Klicken Sie auf Datei > Neu > BAR-Datei.
    2. Geben Sie den Namen der Anwendung ein oder wählen Sie eine Anwendung in der angezeigten Liste aus. Die Liste ist gefiltert, sodass nur Projekte des aktiven Arbeitssets angezeigt werden.
    3. Geben Sie einen Namen für die BAR-Datei ein, die erstellt werden soll.
    4. Klicken Sie auf Fertigstellen. Es wird eine Datei mit der Erweiterung .bar erstellt und in der Ansicht 'Anwendungsentwicklung' im Ordner BARs angezeigt. Der Editor für BAR-Dateien wird geöffnet.
    5. Wählen Sie auf der Registerkarte 'Prepare' (Vorbereiten) die generierte Anwendung aus, die der BAR-Datei hinzugefügt werden soll.
    6. Wählen Sie Ressourcen kompilieren und integrieren aus und klicken Sie anschließend auf Build and Save (Erstellen und speichern), um die BAR-Datei zu erstellen.
  2. Konfigurieren Sie die BAR-Datei. Vor der Implementierung der Nachrichtenflüsse für Ihre Musterinstanz Healthcare: Web-Service an DICOM ist keine weitere Konfiguration des Nachrichtenflusses in der BAR-Datei erforderlich, aber unter Umständen möchten Sie einige der Standardoptionen überprüfen und Änderungen für Ihre Umgebung konfigurieren. Führen Sie im Editor für BAR-Dateien auf der Registerkarte "Verwalten" die folgenden Aktionen aus:
    1. Optional: Über die benutzerdefinierte Eigenschaft LoggingOn können Sie festlegen, ob die Protokollierung für den Web-Service-Nachrichtenfluss aktiviert oder inaktiviert werden soll.
    2. Optional: Über die benutzerdefinierte Eigenschaft ErrorLoggingOn können Sie angeben, ob Fehlernachrichten generiert werden sollen.
    3. Optional: Sie können alle konfigurierbaren Eigenschaften (beispielsweise für die Sicherheit oder die Erstellung weiterer Instanzen) in der BAR-Datei ändern.

      Sie müssen sicherstellen, dass die von Ihnen geänderten Werte sich nicht auf das Verhalten der Musterinstanz auswirken.

  3. Erstellen Sie Warteschlangen. Die WebSphere MQ-Warteschlangen für die Fehler- und Protokollnachrichten müssen erstellt werden, falls sie noch nicht vorhanden sind.
    1. Erstellen Sie WebSphere MQ-Warteschlangen. WebSphere MQ-Ressourcen können entweder über die Befehlszeile von Programmen und Dienstprogrammen wie runmqsc oder über die grafische Schnittstelle von WebSphere MQ erstellt werden. Weitere Informationen zum Erstellen von WebSphere MQ-Ressourcen finden Sie in der Produktdokumentation zu WebSphere MQ auf der Webseite mit der WebSphere MQ-Bibliothek.
  4. Implementieren Sie die BAR-Datei auf einem Integrationsserver.

Nachdem Sie eine BAR-Datei mit Ihrem Muster erstellt haben, können Sie das Muster mit der DICOM-Testanwendung testen. Mit der DICOM-Testanwendung können Sie DICOM-Nachrichten senden und empfangen und Web-Service-Nachrichten senden und empfangen. Weitere Informationen zur Verwendung der DICOM-Testanwendung finden Sie im Abschnitt "DICOM-Testanwendung verwenden" in der Dokumentation zu IBM Integration Bus Healthcare Pack.
