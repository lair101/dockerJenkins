Parameters for the Patient Identifier Cross-reference Manager pattern

Connection Details
Pattern Parameter Default Description
Connection details localhost:1111 This parameter defines the TCP/IP connection details of the Patient Identifier Cross-reference Manager. Configure your clinical application that is acting as the Patient Identity Source in the PIX profile to connect to this address. This parameter is in the form hostname:port
MDM Patient Hub (EMPI)
Pattern Parameter Default Description
Username This parameter specifies the username that is required to connect to the EMPI.
Password This parameter specifies the password that is required to connect to the EMPI.
Web Service URL http://localhost:9080/ibminitiatews/services/IdentityHub This parameter specifies the web service URL that is required to connect to the EMPI identity hub, for example: http://localhost:9080/ibminitiatews/services/IdentityHub
PIX Manager
Pattern Parameter Default Description
Application This parameter defines the Sending Application value that is entered in the MSH segment of acknowledgments that are sent from the Patient Identifier Cross-reference Manager.
Facility This parameter defines the Sending Facility value that is entered in the MSH segment of acknowledgments that are sent from the Patient Identifier Cross-reference Manager.
Namespace ID This parameter defines the namespace ID of the assigning authority (domain) of the Patient Identifier Cross-reference Manager. This parameter is the equivalent of srcCode in the EMPI.

In an incoming ADT message, if the value of the PID.3.4.1 field does not match the value of this parameter, the message is rejected. In an incoming ADT message, if PID.3.4.1 is empty, but PID.3.4.2 and PID.3.4.3 match their corresponding pattern parameters (Universal ID and Universal ID Type), PID.3.4.1 is populated with the value of this parameter.
Universal ID This parameter defines the universal ID of the assigning authority (domain) of the Patient Identifier Cross-reference Manager.

In an incoming ADT message, if the value of the PID.3.4.2 field does not match the value of this parameter, the message is rejected.
Universal ID Type ISO This parameter defines the universal ID type of the assigning authority (domain) of the Patient Identifier Cross-reference Manager.

In an incoming ADT message, if the value of the PID.3.4.3 field does not match the value of this parameter, the message is rejected.
Patient Identity Source
Pattern Parameter Default Description
Application This parameter defines the expected Sending Application value of the MSH.3 field of incoming ADT messages.

If the value in the ADT message does not match the value defined for this parameter, the incoming message is rejected.
Facility This parameter defines the expected Sending Facility value of the MSH.4 field of incoming ADT messages.

If the value in the ADT message does not match the value defined for this parameter, the incoming message is rejected.

All the pattern parameters are listed with their default values. Where no default is indicated, the default value is an empty string.
