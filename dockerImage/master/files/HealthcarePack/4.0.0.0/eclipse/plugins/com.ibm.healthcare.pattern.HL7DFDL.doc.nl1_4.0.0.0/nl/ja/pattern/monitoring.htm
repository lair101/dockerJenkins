「Healthcare: HL7 to HL7 DFDL」パターンのモニター

「Healthcare: HL7 to HL7」パターンによって生成されるフローは、Web ベースのモニターでモニターされます。 モニター・ビューには、メッセージ・フローによってパブリッシュされる TCP/IP 統計が表示されます。 この統計は、メッセージ・フローにあるユーザー定義プロパティーに格納された情報で補足されます。 このトピックでは、これらのユーザー定義プロパティーの名称と、それに含まれる情報について説明します。

パターンの外側で HL7 ノードを使用してカスタム統合を作成している場合に、メッセージ・フローをモニター・ビューと統合するには、同じユーザー定義プロパティーを構成します。

ユーザー定義プロパティー

このパターンは、常に 1 つの受信側フローと、1 つ以上の宛先を持ちます。 そのため、ソース・アプリケーションの ID とファシリティーは、すべての宛先に対して固定されます。 宛先の ID とファシリティーは、どの宛先メッセージ・フローを検査するかに応じて異なります。

プロパティー名 値 例
Identifier1 パターン・インスタンス名 ADT
Identifier2 これが受信側と送信側のどちらのメッセージ・フローかを示すリテラル・ストリング Source または Destination
Identifier3 これが受信側と送信側のどちらのメッセージ・フローかであるかに応じた、ソース・アプリケーションまたは宛先アプリケーションの HL7 ID。 Patient Registration
Identifier4 これが受信側と送信側のどちらのメッセージ・フローであるかに応じた、ソース・アプリケーションまたは宛先アプリケーションの HL7 ファシリティー West City Hospital
TCPIPNode メッセージ・フローによって使用される HL7 ノードのノード名と TCP/IP アドレス HL7 In/localhost:1111 または HL7 Out/localhost:2222
SourceIdentifier ソース・アプリケーションの HL7 ID Patient Registration
SourceFacility ソース・アプリケーションの HL7 ファシリティー West City Hospital
DestinationIdentifier 宛先アプリケーションの HL7 ID Pharmacy
DestinationFacility 宛先アプリケーションの HL7 ファシリティー Another City Hospital
QueuePrefix このメッセージ・フローのキュー接頭部、またはブランク。 キュー接頭部としてブランクを指定すると、パターン・インスタンス名がキュー接頭部として使用されます。 ADTQ
