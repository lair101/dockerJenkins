Parámetros para el patrón de HL7 a DFDL HL7

Origen
Parámetro de patrón Valor predeterminado Descripción
Tipo de origen Genérico Este parámetro define el tipo de aplicación de origen. El parámetro se utiliza para seleccionar el nodo HL7DFDLInput en el flujo Receiver.
Recurso de origen Este parámetro define el recurso de origen que se incluye en los mensajes de error, notificaciones y mensajes de registro estándar.
Identificador de origen Aplicación de origen Este parámetro define un identificador que se utiliza para describir el origen en los mensajes de error, notificaciones y mensajes de registro estándar.
Información detallada de conexión localhost:1111 Este parámetro define la conexión TCP/IP para la aplicación de origen con el formato nombre_host:puerto
Tiempo de espera de datos (segundos) 60 Este parámetro define el tiempo que va a estar a la escucha un nodo en una conexión a la espera de más datos después de la llegada del primer byte de datos. Puede especificar cualquier cantidad de tiempo en segundos. El valor predeterminado es 60 segundos. Cuando se excede el tiempo especificado, todos los datos disponibles se envían al terminal de anomalías (Failure).
Byte MLLP inicial 0B Este parámetro define el byte MLLP inicial que se debe quitar de la parte frontal del registro HL7.
Bytes MLLP finales 1C0D Este parámetro define los bytes MLLP finales que se utilizan como un delimitador de registros HL7. Estos bytes MLLP finales los elimina el nodo TCPIPServerInput. También los añade el nodo TCPIPClientOutput que devuelve los acuses de recibo de HL7.
Límite de errores consecutivos 0 Este parámetro define el número máximo de errores consecutivos que se toleran antes de cerrar la conexión de red. Un número no positivo se trata como un valor infinito y la conexión no se finaliza si hay errores consecutivos.

La conexión siempre se finaliza si el mensaje no contiene el byte MLLP inicial correcto.
Destinos
Parámetro de patrón Valor predeterminado Descripción
Número de destinos 1 Este parámetro define el número de destinos a los que se distribuyen los mensajes de origen.
Parte 1 Este parámetro define qué parte de un patrón de varias partes se está configurando actualmente.

Para obtener más información sobre cómo configurar varias instancias de patrón, consulte Ejecución del patrón.
Parámetros de destino
Parámetro de patrón Valor predeterminado Descripción
Tipo de destino Genérico Este parámetro define el tipo de aplicación de destino. El parámetro se utiliza para seleccionar el nodo HL7DFDLOutput en el flujo Sender.
Identificador Destino 1 Este parámetro define la identidad de un destino. Esta identidad se utiliza en los mensajes de error, de informe y de notificación relacionados con un destino
Recurso de destino Este parámetro define el recurso de un destino. Este recurso se utiliza en los mensajes de error, de informe y de notificación relacionados con un destino
Acción de secuenciación Ninguna Si se selecciona la opción de aplicar números de secuencia en el flujo Receiver, este parámetro determina cómo se aplica la secuenciación para un destino específico. Los valores válidos son:
  • Ninguno: Los mensajes se procesan en el orden en que llegan.
  • Estricto: Los mensajes no se pueden procesar si no es en orden. Si se pierde un mensaje, se retienen todos los demás mensajes hasta que llega el que se ha perdido.
  • Laxo: En general, los mensajes se procesan en orden. Sin embargo, si un mensaje no llega en un tiempo especificado, se procesan los demás mensajes. Si el mensaje que falta llega después del tiempo de espera, se inserta en la corriente.
Tiempo de espera de secuenciación (segundos) Este parámetro define el tiempo, en segundos, que el flujo de mensajes espera un mensaje que no llega en la secuencia correcta. Después de este tiempo, los mensajes siguen su flujo. Si los mensajes que faltan llegan más tarde, se insertan en la corriente.
Filtrado de mensajes Borrado Este parámetro determina si los mensajes que se envían a un destino se filtran. Cuando este parámetro está seleccionado, la tabla de filtrado de mensajes está visible y se pueden especificar criterios de filtrado.

El filtrado garantiza que sólo pasan los mensajes donde los valores de MSH-9 (campo Tipo de mensaje) para Código de mensaje y Suceso desencadenante coincidan con los valores de Filtro 1 y Filtro 2.

Si se requiere un filtrado más complejo, puede modificar el código ESQL con el mismo método.
Filtrado de segmentos Borrado Este parámetro determina, para un destino, los segmentos que se copian desde el mensaje de origen. Cuando este parámetro está seleccionado, la tabla de filtrado de segmentos está visible y se pueden especificar criterios de filtrado.
Información detallada de conexión localhost:2222 Este parámetro define la conexión TCP/IP para un destino con el formato nombre_host:puerto
MLLP estándar Seleccionado Este parámetro determina si se utiliza una configuración predeterminada para TCP/IP y MLLP. La configuración predeterminada tiene los siguientes valores:
  • Tiempo de espera: 60 segundos
  • Byte MLLP inicial: 0B
  • Delimitador: No aplicable
  • Límite de reintentos: 3
  • Registrar de reintentos: true
  • Validación: false

Si el parámetro no está seleccionado, se hace visible el grupo de parámetros para definir valores no estándar.
Filtros de mensajes para el destino
Esta tabla define el conjunto de mensajes que se pasan a la aplicación de destino.
Parámetro de la tabla Obligatorio Valor predeterminado Descripción
Código Sí   Este parámetro define un código de mensaje con el que un mensaje debe coincidir para pasarlo al destino.
Suceso Sí   Este parámetro define los sucesos de mensaje que se utilizan para filtrar mensajes. Si se admiten todos los sucesos de mensaje, introduzca un asterisco (*).
Filtros de segmentos para el destino
Esta tabla define el conjunto de segmentos que se pasan a la aplicación de destino.
Parámetro de la tabla Obligatorio Valor predeterminado Descripción
Segmento Sí   Este parámetro define los códigos de segmento que se utilizan para filtrar segmentos. Si el código de segmento de tres letras coincide con alguna entrada de la tabla de filtros de segmentos, el segmento se excluye del mensaje de salida.
Comunicaciones de destino
Parámetro de patrón Valor predeterminado Descripción
Tiempo de espera (segundos) 60 Este parámetro define el tiempo de espera para un destino en segundos.
Byte MLLP inicial 0B Este parámetro define el byte MLLP inicial, que se añade a los registros de salida.
Bytes MLLP finales 1C0D Este parámetro define el delimitador. Este valor debe ser hexadecimal.
Límite de reintentos 3 Este parámetro define el número de veces que un flujo Sender intenta entregar un mensaje HL7 a la aplicación de destino.
Registrar reintento Seleccionado Este parámetro determina si se registra cada uno de los intentos de entregar un mensaje.
Validación Ninguna Este parámetro determina si el mensaje se valida frente al conjunto de mensajes HL7.
Opciones
Parámetro de patrón Valor predeterminado Descripción
Canal de información de origen Seleccionado Este parámetro determina si se genera una copia del mensaje de origen. El parámetro Publicar determina si el canal de información de origen se graba en la cola instance.SRCF o se publica en el tema PatternId/Receiver/Source
Números de secuencia Basado en llegada Este parámetro determina cómo se definen los números de secuencia de los mensajes para el comportamiento definido por el parámetro de secuenciación de cada destino. Los valores válidos son:
  • Sin secuenciación
  • Basado en llegada: Los números de secuencia se asignan según el orden de llegada al nodo.
  • Basado en contenido: Los números de secuencia se definen a partir del número de secuencia que se encuentra en la cabecera MSH.
Enviar acuse de recibo Seleccionado Este parámetro determina si la aplicación emisora espera una respuesta ACK o NACK estándar.
Registrar por diario Estándar Este parámetro determina las opciones de registro por diario. Los valores válidos son:
  • Ninguno: No registrar por diario. Esta opción también se puede utilizar si el usuario del patrón quiere utilizar un canal de información de origen para llevar a cabo el registro por diario para sus propios requisitos.
  • Estándar: Se genera una copia de cada mensaje con la información de cabecera estándar definida por el patrón. Esta copia se publica en el tema PatternId/Receiver/Journal o se escribe en la cola de diario según determine el parámetro Publish.
Canal de información canónico Borrado Este parámetro determina si se genera una copia del mensaje en formato canónico. Si se selecciona, el parámetro Publicar determina si esta copia del mensaje se envía a la cola o PatternId.CAN o al tema PatternId/Canonical
Informar de restos Seleccionado Este parámetro determina si se informa de los campos restantes.

Si un mensaje entrante tiene segmentos que no están modelados en el conjunto de mensajes HL7, se encuentran los campos restantes.

Si este parámetro está seleccionado, el parámetro Publicar determina si los mensajes con campos restantes se envían a la cola PatternId.REM o se publican.

Esta acción es especialmente útil durante el desarrollo de la integración entre aplicaciones HL7.
Comprobar duplicados Borrado Este parámetro determina si se detectan los mensajes duplicados, según determine el identificador de mensaje exclusivo en la cabecera MSH de HL7.

Si se detectan duplicados se lleva a cabo la siguiente acción:

  • Si se recibe el mensaje dentro del periodo de espera, el mensaje no se procesa pero se devuelve al remitente el mismo acuse de recibo enviado para el mensaje original.
  • Una vez transcurrido un periodo de tiempo, los mensajes duplicados se procesan como de costumbre.
Publicar Publicar Este parámetro determina si los mensajes de diario, los mensajes restantes y otros informes se publican o se graban en colas.
Separar colas de secuencias Borrado Este parámetro determina si se utilizan colas separadas para cada instancia de patrón para almacenar mensajes durante la secuenciación. De forma predeterminada, todos los mensajes almacenados para la secuenciación en un nodo de integración se retienen en una única cola del sistema.

Si se selecciona este Si este parámetro está seleccionado, se define un servicio configurable para esta instancia de patrón. Este servicio configurable se utiliza para definir las colas internas que utilizan los nodos Resequence en los flujos Sender.
Procesamiento de duplicados
Parámetro de patrón Valor predeterminado Descripción
Informar de duplicados Seleccionado Este parámetro determina si los mensajes duplicados que llegan dentro del tiempo especificado generan un mensaje de notificación de error.
Caducidad de mensajes de identificador (segundos) 86400 Este parámetro define (en segundos) durante cuánto tiempo se conservan los mensajes de identificador antes de que caduquen. Una vez transcurrido este tiempo, los duplicados no se reconocen y se procesan como de costumbre. El valor predeterminado es 24 horas.
General
Parámetro de patrón Valor predeterminado Descripción
Descripción Patrón HL7 Este parámetro define la finalidad del flujo que se genera y puede incluir cualquier información detallada que desee.
Prefijo de cola Este parámetro define la serie de caracteres que desea prefijar a los nombres de cola. Las colas estándar se crean con este prefijo y un conjunto estándar de sufijos.

Si este parámetro se deja en blanco, se procesa el nombre de instancia de patrón eliminando los caracteres no alfanuméricos y se utiliza como prefijo de cola.
Prefijo de flujo Este parámetro define una serie de caracteres que se añade al principio del nombre de cada flujo de mensajes principal en una instancia de patrón generada. Esta serie le ayuda a identificar los componentes de la instancia de patrón en los informes y en la supervisión.

Si este parámetro se deja en blanco, se procesa el nombre de instancia de patrón eliminando los caracteres no alfanuméricos y se utiliza como prefijo del flujo.
Generar scripts Seleccionado Este parámetro determina si se genera un script de despliegue MQSC.
Nombre de archivo de rastreo tracefile.txt Este parámetro define el nombre del archivo, incluyendo la vía de acceso, al que se envía la salida de rastreo. Si no se indica ninguna vía de acceso, el archivo de rastreo se graba en el directorio de vía de acceso de trabajo para toda la máquina de la instalación de IBM Integration Bus. En Windows, el directorio de vía de acceso de trabajo predeterminado es C:\ProgramData\Application Data\IBM\MQSI. En Linux, el directorio de vía de acceso de trabajo predeterminado es /var/mqsi.

Todos los parámetros de patrón aparecen en la lista con sus valores predeterminados. Cuando no se indica un valor predeterminado, éste es una serie vacía.
