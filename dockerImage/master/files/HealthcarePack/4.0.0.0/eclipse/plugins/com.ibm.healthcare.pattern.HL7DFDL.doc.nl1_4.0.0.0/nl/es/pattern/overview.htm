Patrón Servicio sanitario: HL7 a DFDL HL7

El patrón Servicio sanitario: HL7 a DFDL HL7 integra una aplicación que puede enviar mensajes HL7 (Health Level Seven International) v2 con una o más aplicaciones que pueden recibir mensajes HL7. Las aplicaciones deben poder enviar y recibir mensajes HL7 utilizando el protocolo MLLP (Minimal Lower Layer Protocol) sobre TCP/IP.

Importante

Debe importar una biblioteca DFDL HL7 en su espacio de trabajo. El modelo DFDL de la biblioteca se utiliza para analizar los mensajes HL7 a medida que los procesan los flujos de mensajes. Debe incluir una referencia a la biblioteca DFDL HL7 en su proyecto de forma que la biblioteca DFDL se puede desplegar juntamente con la aplicación generada. De forma predeterminada, la aplicación que genera el patrón contiene una referencia a la biblioteca que contiene el modelo DFDL de HL7 v2.7. Si desea utilizar los modelos DFDL de HL7 v2.6 o HL7 v2.5.1, consulte Preparación de la biblioteca DFDL HL7.

Puede importar la biblioteca DFDL HL7 en el espacio de trabajo desde la página Recursos para el patrón.

La biblioteca DFDL HL7 tarda mucho tiempo en construirse y desplegarse. Puede reducir el tiempo que se tarda en construir el modelo DFDL suprimiendo los capítulos de HL7 del conjunto de mensajes. Conserve los archivos de definición de mensajes GenericHL7.xsd, segments.xsd, fields.xsd y datatypes.xsd en el conjunto de mensajes, ya que los necesita el patrón.

Opciones

El flujo Receiver de este patrón admite les siguientes opciones:

  • Recepción de mensajes HL7 utilizando MLLP sobre TCP/IP
  • Validación y análisis de los mensajes
  • Auditoría de los mensajes entrantes
  • Comprobación y proceso de mensajes duplicados
  • Soporte a las opciones de secuenciación
  • Publicación de cada uno de los mensajes de entrada o envío del mensaje a una cola para su proceso fuera del patrón
  • Envío de la salida a una cola de transformación y direccionamiento
  • Manejo de excepciones

El flujo TransformAndRoute de este patrón admite las siguientes opciones:

  • Transformación a formato XML canónico
  • Publicación del mensaje canónico o envío del mensaje a una cola para su proceso fuera del patrón
  • Informes de los campos adicionales encontrados en los segmentos HL7
  • Distribución de los mensajes a diversos destinos, configurables por separado
  • Filtrado de los mensajes a enviar a cada destino
  • Filtrado de los segmentos a incluir para cada destino
  • Puntos de personalización para habilitar la inclusión de cambios no estándar específicos de cada destino
  • Volver a transformar a HL7 y envío de la salida a la cola del flujo Sender de cada destino
  • Manejo de excepciones

Cada uno de los flujos Sender de este patrón admite las siguientes opciones:

  • Lectura de mensajes transformados para este destino
  • Validación de los mensajes transformados
  • Garantizar que los mensajes se entregan secuencialmente según las opciones de secuenciación
  • Notificar cuando los mensajes llegan fuera de secuencia
  • Entrega de los mensajes HL7 a la aplicación de destino utilizando MLLP sobre TCP/IP
  • Manejo de excepciones
