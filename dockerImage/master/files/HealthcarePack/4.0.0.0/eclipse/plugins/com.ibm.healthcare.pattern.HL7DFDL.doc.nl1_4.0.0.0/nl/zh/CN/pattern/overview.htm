“Healthcare：从 HL7 到 HL7 DFDL”模式

“Healthcare：从 HL7 到 HL7 DFDL”模式将能够发送 Health Level Seven International (HL7) V2 消息的应用程序与一个或多个能够接收 HL7 消息的应用程序集成。 这些应用程序必须能够使用基于 TCP/IP 的 Minimal Lower Layer Protocol (MLLP) 发送和接收 HL7 消息。

重要事项！

必须将 HL7 DFDL 库导入到工作空间中。这个库中的 DFDL 模型用于在消息流处理 HL7 消息时对这些消息进行解析。必须在项目中包含对 HL7 DFDL 库的引用，以便该 DFDL 库可以与生成的应用程序一起部署。缺省情况下，此模式生成的应用程序包含对 HL7 V2.7 DFDL 模型所在的库的引用。如果希望使用 HL7 V2.6 或 HL7 V2.5.1 DFDL 模型，请参阅准备 HL7 DFDL 库。

您可以从模式的资源页面将 HL7 DFDL 库导入到工作空间中。

构建和部署 HL7 DFDL 库需要花费较长时间。通过从消息集中删除 HL7 章节，可以缩短构建 DFDL 模型所需的时间。 将 GenericHL7.xsd、segments.xsd、fields.xsd 和 datatypes.xsd 消息定义文件保留在消息集中，因为这些是模式所需的文件。

选项

此模式中的“接收方”流支持下列选项：

  • 使用基于 TCP/IP 的 MLLP 接收 HL7 消息
  • 验证并解析消息
  • 审计入局消息
  • 检查并处理重复消息
  • 支持排序选项
  • 发布每条入局消息，或者将该消息发送到队列以便在模式外进行处理
  • 将输出发送到“变换并路由”队列
  • 处理异常

此模式中的 TransformAndRoute 流支持下列选项：

  • 变换为规范 XML 格式
  • 发布规范入局消息，或者将该消息发送到队列以便在模式外进行处理
  • 对 HL7 段中的其他字段进行报告
  • 将消息分发到多个可分别配置的目标
  • 对要发送到各个目标的消息进行过滤
  • 对要针对各个目标包括的段进行过滤
  • 定制点，用于支持包括特定于目标的非标准更改
  • 重新变换为 HL7，并将输出发送到各个目标的“发送方”流的队列
  • 处理异常

此模式中的每个“发送方”流支持下列选项：

  • 为此目标读取变换后的消息
  • 对变换后的消息进行验证
  • 确保根据排序选项按顺序交付消息
  • 在消息未按顺序到达时发出通知
  • 使用基于 TCP/IP 的 MLLP 将 HL7 消息交付到目标应用程序
  • 处理异常
