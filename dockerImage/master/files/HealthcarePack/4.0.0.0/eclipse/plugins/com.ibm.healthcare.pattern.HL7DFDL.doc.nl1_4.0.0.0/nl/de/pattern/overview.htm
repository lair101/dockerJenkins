Muster 'Healthcare: HL7 an HL7 DFDL'

Das Muster Healthcare: HL7 an HL7 DFDL integriert eine Anwendung, die HL7-v2-Nachrichten (Health Level Seven International) senden kann, in eine oder mehrere Anwendungen, die HL7-Nachrichten empfangen können. Die Anwendungen müssen in der Lage sein, HL7-Nachrichten unter Verwendung des MLLP-Protokolls (MLLP = Minimal Lower Layer Protocol) über TCP/IP zu senden und zu empfangen.

Wichtig:

Sie müssen eine HL7-DFDL-Bibliothek in Ihren Arbeitsbereich importieren. Mit dem DFDL-Modell in der Bibliothek werden HL7-Nachrichten bei der Verarbeitung in den Nachrichtenflüssen analysiert. Ihr Projekt muss einen Verweis auf die HL7-DFDL-Bibliothek enthalten, damit die DFDL-Bibliothek zusammen mit der generierten Anwendung implementiert werden kann. Standardmäßig enthält die vom Muster generierte Anwendung einen Verweis auf die Bibliothek, die das HL7-DFDL-Modell Version 2.7 enthält. Wenn Sie das HL7-DFDL-Modell der Version 2.6 oder 2.5.1 verwenden möchten, lesen Sie den Abschnitt HL7-DFDL-Bibliothek vorbereiten.

Sie können die HL7-DFDL-Bibliothek über die Seite Ressourcen für das Muster 'Healthcare: HL7 an HL7 DFDL' in Ihren Arbeitsbereich importieren.

Die Erstellung und Implementierung der HL7-DFDL-Bibliothek nimmt einige Zeit in Anspruch. Sie können die Erstellung des DFDL-Modells beschleunigen, indem Sie die HL7-Kapitel aus der Nachrichtengruppe löschen. Die Nachrichtendefinitionsdateien GenericHL7.xsd, segments.xsd, fields.xsd und datatypes.xsd müssen in der Nachrichtengruppe verbleiben, da sie vom Muster benötigt werden.

Optionen

Der Nachrichtenfluss 'Receiver' in diesem Muster unterstützt die folgenden Optionen:

  • Empfang von HL7-Nachrichten unter Verwendung von MLLP über TCP/IP
  • Validierung und Parsing der Nachrichten
  • Überwachung eingehender Nachrichten
  • Überprüfung und Verarbeitung doppelter Nachrichten
  • Unterstützung von Sequenzbildungsoptionen
  • Veröffentlichung eingehender Nachrichten bzw. Senden der Nachrichten an eine Warteschlange zur Weiterverarbeitung außerhalb des Musters
  • Senden der Ausgabe an eine Warteschlange zur Transformation und Weiterleitung
  • Behandlung von Ausnahmebedingungen

Der Nachrichtenfluss 'TransformAndRoute' in diesem Muster unterstützt die folgenden Optionen:

  • Transformation in ein kanonisches XML-Format
  • Veröffentlichung eingehender kanonischer Nachrichten bzw. Senden der Nachrichten an eine Warteschlange zur Weiterverarbeitung außerhalb des Musters
  • Auflisten zusätzlicher Felder, die in HL7-Segmenten gefunden werden
  • Verteilung von Nachrichten an mehrere, gesondert konfigurierbare Ziele
  • Filtern der an die einzelnen Ziele zu sendenden Nachrichten
  • Filtern der Segmente, die für jedes Ziel einbezogen werden sollen
  • Anpassungspunkte, damit zielspezifische Änderungen einbezogen werden können, die vom Standard abweichen
  • Transformation in HL7 und Senden der Ausgabe an die Warteschlange für den Nachrichtenfluss 'Sender' für jedes Ziel
  • Behandlung von Ausnahmebedingungen

Jeder Sender-Nachrichtenfluss in diesem Muster unterstützt die folgenden Optionen:

  • Lesen transformierter Nachrichten für dieses Ziel
  • Validierung transformierter Nachrichten
  • Sicherstellen, dass Nachrichten in Übereinstimmung mit den Sequenzbildungsoptionen der Reihe nach zugestellt werden
  • Benachrichtigung, wenn Nachrichten in falscher Reihenfolge eintreffen
  • Zustellung von HL7-Nachrichten an die Zielanwendung unter Verwendung von MLLP über TCP/IP
  • Behandlung von Ausnahmebedingungen
