监视“Healthcare：从 HL7 到 HL7 DFDL”模式

“Healthcare：从 HL7 到 HL7”模式所生成的流通过基于 Web 的监视进行监视。此监视视图显示消息流发布的 TCP/IP 统计信息。这些统计信息根据消息流中用户定义属性中存储的信息进行扩充。本主题对这些用户定义属性的名称及其包含的信息进行说明。

如果您是使用此模式外部的 HL7 节点来创建定制集成，那么可以通过配置相同的用户定义属性将消息流与监视视图集成。

用户定义属性

此模式始终具有单个“接收方”以及一个或多个目标。因此，对于所有目标，源应用程序标识和设施固定不变。目标标识和设施将随您检查的目标消息流不同而有所变化。

属性名 值 示例
Identifier1 模式实例名称 ADT
Identifier2 文字串，具体取决于此流是“接收方”还是“发送方”消息流 源或目标
Identifier3 源或目标应用程序 HL7 标识，具体取决于此流是“接收方”还是“发送方”消息流 患者注册
Identifier4 源或目标应用程序 HL7 设施，具体取决于此流是“接收方”还是“发送方”消息流 西城区医院
TCPIPNode 消息流所使用的 HL7 节点的节点名和 TCP/IP 地址 HL7 In/localhost:1111 或 HL7 Out/localhost:2222
SourceIdentifier 源应用程序 HL7 标识 患者注册
SourceFacility 源应用程序 HL7 设施 西城区医院
DestinationIdentifier 目标应用程序 HL7 标识 药房
DestinationFacility 目标应用程序 HL7 设施 其他城区医院
QueuePrefix 此消息流的队列前缀或空白 - 空白队列前缀表示将使用模式实例名称作为队列前缀 ADTQ
