「Healthcare: HL7 to HL7 DFDL」パターン

「Healthcare: HL7 to HL7 DFDL」パターンは、Health Level 7 International (HL7) v2 メッセージを送信できるアプリケーションを、HL7 メッセージを受信できる 1 つ以上のアプリケーションと統合します。 それらのアプリケーションは、Minimal Lower Layer Protocol (MLLP) over TCP/IP を使用して HL7 メッセージを送信および受信できる必要があります。

重要

HL7 DFDL ライブラリーをワークスペースにインポートする必要があります。 HL7 メッセージがメッセージ・フローによって処理される際には、このライブラリー内の DFDL モデルを使用して HL7 メッセージが解析されます。 生成したアプリケーションと一緒に DFDL ライブラリーを実装できるようにするために、プロジェクトに HL7 DFDL ライブラリーへの参照を組み込む必要があります。デフォルトでは、パターンによって生成されるアプリケーションには、HL7 v2.7 DFDL モデルを含むライブラリーへの参照が組み込まれます。 HL7 v2.6 または HL7 v2.5.1 の DFDL モデルを使用する場合は、「HL7 DFDL ライブラリーの準備」を参照してください。

HL7 DFDL ライブラリーは、「パターンのリソース」ページからワークスペースにインポートできます。

HL7 DFDL ライブラリーは、ビルドとデプロイに長時間を要します。 DFDL モデルのビルドにかかる時間を短縮するには、メッセージ・セットから HL7 チャプターを削除します。 GenericHL7.xsd、segments.xsd、fields.xsd、datatypes.xsd の各メッセージ定義ファイルは、パターンで必要になるので、メッセージ・セットにそのまま残しておきます。

オプション

このパターンに含まれる受信側フローでは、以下のオプションがサポートされます。

  • MLLP over TCP/IP を使用して HL7 メッセージを受信する。
  • メッセージを検証および解析する。
  • 着信メッセージを監査する。
  • 重複したメッセージをチェックし、処理する。
  • 順序付けオプションをサポートする。
  • 各着信メッセージをパブリッシュする。 またはパターンの外側で処理するためにメッセージをキューに送信する。
  • 変換および経路指定キューに出力を送信する。
  • 例外を処理する。

このパターンに含まれる TransformAndRoute フローでは、以下のオプションがサポートされます。

  • Canonical XML フォーマットに変換する。
  • 着信する正規メッセージをパブリッシュする。 またはパターンの外側で処理するためにメッセージをキューに送信する。
  • HL7 セグメントに含まれる追加のフィールドについてレポートする。
  • メッセージを、独立して構成可能な複数の宛先に配布する。
  • メッセージをフィルタリングして、各宛先に送信する。
  • セグメントをフィルタリングして、各宛先に組み込む。
  • 非標準で宛先に固有の変更を組み込めるようにするカスタマイズ・ポイント。
  • HL7 に逆変換し、各宛先の送信側フロー用のキューに出力を送信する。
  • 例外を処理する。

このパターンに含まれるそれぞれの送信側フローでは、以下のオプションがサポートされます。

  • この宛先用に変換されたメッセージを読み取る。
  • 変換されたメッセージを妥当性検査する。
  • 順序付けオプションに従ってメッセージが順番に送達されるようにする。
  • 順番どおりに到着しなかったメッセージがあれば通知する。
  • MLLP over TCP/IP を使用して HL7 メッセージ宛先アプリケーションに送達する。
  • 例外を処理する。
