“从 HL7 到 HL7 DFDL”模式的参数

源
模式参数 缺省值 描述
源类型 通用 此参数定义源应用程序的类型。此参数用于选择“接收方”流中的 HL7DFDLInput 节点。
源设施 此参数定义要包括在错误消息、通知和标准日志记录消息中的源设施。
源标识 源应用程序 此参数定义一个标识，用于在错误消息、通知和标准日志记录消息中描述源。
连接详细信息 localhost:1111 此参数以 hostname:port 格式定义源应用程序的 TCP/IP 连接。
等待数据超时（秒） 60 此参数定义第一个数据字节到达后，节点侦听连接以获取更多数据的时间长度。您可以指定以秒为单位的任意时间长度。缺省值为 60 秒。超出指定时间后，所有可用数据都将发送到 Failure 终端。
前导 MLLP 字节 0B 此参数定义在 HL7 记录前必须修剪掉的前导 MLLP 字节。
结尾 MLLP 字节 1C0D 此参数定义用作 HL7 记录定界符的结尾 MLLP 字节。TCPIPServerInput 节点将除去这些结尾 MLLP 字节。另外，返回 HL7 应答的 TCPIPClientOutput 节点也将添加这些字节。
连续错误限制 0 此参数定义能够容忍的最大连续错误数，超过此数目后，将关闭网络连接。任何非正数都将被视为无穷大值，在这种情况下，连续发生故障并不会导致连接结束。

在消息未包含正确的前导 MLLP 字节时，连接始终结束。
目标
模式参数 缺省值 描述
目标数 1 此参数定义源消息将分发到的目标数。
部分 1 此参数定义多部分模式中当前所配置的部分。

有关配置多部分模式实例的更多信息，请参阅运行模式。
目标参数
模式参数 缺省值 描述
目标类型 通用 此参数定义目标应用程序的类型。此参数用于选择“发送方”流中的 HL7DFDLOutput 节点。
标识 目标 1 此参数定义目标的身份。在与目标相关的错误、报告和通知消息中，将使用此身份。
目标设施 此参数定义目标的设施。在与目标相关的错误、报告和通知消息中，将使用此设施。
后续操作 无 如果在“接收方”流中选择了应用序号的选项，那么此参数确定如何对目标进行排序。有效值包括：
  • 无：按消息到达顺序对其进行处理。
  • 严格：必须按顺序处理消息。如果缺少某条消息，那么在此消息到达之前，所有其他消息将挂起。
  • 宽松：一般情况下按顺序处理消息。但是，如果某条消息未在指定的时间段内到达，那么将处理其他消息。如果缺少的消息在超时后到达，那么此消息将插入到流中。
序列超时（秒） 此参数定义消息流等待未按正确顺序到达的消息的时间（以秒计）。在此时间之后，消息将继续流动。如果缺少的消息在稍后到达，那么这些消息将插入到流中。
消息过滤 取消选中 此参数确定是否对发送到目标的消息进行过滤。如果选中此参数，那么消息过滤表将变得可见，并且可以输入过滤条件。

过滤确保只有那些值来自“消息代码”和“触发器事件”的 MSH-9（“消息类型”字段）的消息与“过滤器 1”和“过滤器 2”值匹配。

如果需要进行更复杂的过滤，那么可以按同一方法修改 ESQL 代码。
段过滤 取消选中 对于目标，此参数确定从源消息复制哪些段。如果选中此参数，那么段过滤表将变得可见，并且可以输入过滤条件。
连接详细信息 localhost:2222 此参数以 hostname:port 格式定义目标的 TCP/IP 连接。
标准 MLLP 选中 此参数确定是否使用 TCP/IP 和 MLLP 的缺省配置。缺省配置使用下列值：
  • 超时：60 秒
  • 前导 MLLP 字节：0B
  • 定界符：不适用
  • 重试限制：3
  • 记录重试：true
  • 验证：false

如果未选中此参数，那么用于定义非标准值的参数组将变为可见。
目标的消息过滤器
此表定义要传递到目标应用程序的消息集。
表参数 必需 缺省值 描述
代码 是   此参数定义一个消息代码，消息必须与此消息代码匹配才能传递到目标。
事件 是   此参数定义用于过滤消息的消息事件。如果允许所有消息事件，请输入星号 (*)。
目标的段过滤器
此表定义要传递到目标应用程序的段集。
表参数 必需 缺省值 描述
段 是   此参数定义用于过滤段的段代码。如果三字母段代码与段过滤器表中的任何条目匹配，那么将从出局消息中排除该段。
目标通信
模式参数 缺省值 描述
超时（秒） 60 此参数定义目标的超时值（以秒计）。
前导 MLLP 字节 0B 此参数定义对出局记录添加的前导 MLLP 字节。
结尾 MLLP 字节 1C0D 此参数定义定界符。此值必须采用十六进制格式。
重试限制 3 此参数定义“发送方”流尝试将 HL7 消息交付给目标应用程序的次数。
记录重试 选中 此参数确定是否对每次交付消息的尝试进行记录。
验证 无 此参数确定是否根据 HL7 消息集对消息进行验证。
选项
模式参数 缺省值 描述
源订阅源 选中 此参数确定是否生成源消息的副本。“发布”参数确定源订阅源是将写入 instance.SRCF 队列，还是发布到 PatternId/Receiver/Source 主题。
序列号 根据到达顺序 此参数确定如何针对各个目标的排序参数所定义的排序行为设置消息序号。有效值包括：
  • 不排序
  • 根据到达顺序：根据到达节点的顺序分配序号。
  • 根据内容：根据在 MSH 头中找到的序号分配序号。
发送应答 选中 此参数确定发送应用程序是期望标准 ACK 响应还是 NACK 响应。
日志记录 标准 此参数确定日志记录选项。有效值包括：
  • 无：不进行日志记录。如果模式用户希望使用源订阅源对自己的需求执行日志记录，那么也可以使用此选项。
  • 标准：使用此模式定义的标准头信息生成每条消息的副本。此副本将发布到 PatternId/Receiver/Journal 主题，或者写入由“发布”参数确定的日志队列。
规范订阅源 取消选中 此参数确定是否生成采用规范格式的消息副本。如果选中此参数，那么“发布参数确定此消息副本是发送到 PatternId.CAN 队列还是发送到 PatternId/Canonical 主题。
报告其余项 选中 此参数确定是否报告其余字段。

如果入局消息包含未在 HL7 消息集中建模的段，那么将找到其余字段。

如果选中此参数，那么“发布”参数确定具有其余字段的消息是发送到 PatternId.REM 队列还是进行发布。

开发 HL7 应用程序之间的集成时，此操作尤其有用。
检查重复 取消选中 此参数确定是否检测重复消息，重复与否由 HL7 MSH 头中的唯一消息标识确定。

如果检测到重复，那么将执行下列操作：

  • 如果是在超时时间段内接收到该消息，那么不处理该消息，而是向发送方返回针对原始消息发送的同一 ACK。
  • 在超时时间段到期后，所有重复消息都正常地进行处理。
发布 发布 此参数确定是发布日志消息、其余消息和其他报告，还是将这些消息和报告写入队列。
单独的排序队列 取消选中 此参数确定在排序期间，是否将单独的队列用于各个模式实例以存储消息。缺省情况下，所有存储在集成节点上等待排序的消息将存放在单个系统队列中。

如果选中此参数，那么将为此模式实例定义一个可配置服务。这个可配置服务用于定义“发送方”流中的 Resequence 节点所使用的内部队列。
重复处理
模式参数 缺省值 描述
报告重复 选中 此参数确定在指定时间内到达的重复消息是否生成错误通知消息。
标识消息到期时间（秒） 86400 此参数定义在到期前保留消息标识的时间长度（以秒计）。此时间段过后，将不会识别重复消息，而是正常地对其进行处理。缺省值为 24 小时。
常规
模式参数 缺省值 描述
描述 HL7 模式 此参数定义所生成的流的用途，并且可以包括您需要的任何详细信息。
队列前缀 此参数定义要用作队列名称前缀的字符串。标准队列将使用此前缀和标准后缀集进行创建。

如果将此参数保留为空，那么将对模式实例名进行处理以除去非字母数字字符，然后将处理后的模式实例名用作队列前缀。
流前缀 此参数定义一个字符串，此字符串将添加到生成的模式实例中每个主消息流的名称开头。此字符串可以帮助在报告和监视环境中标识模式实例的组件。

如果将此参数保留为空，那么将对模式实例名进行处理以除去非字母数字字符，然后将处理后的模式实例名用作流前缀。
生成脚本 选中 此参数确定是否生成 MQSC 部署脚本。
跟踪文件名 tracefile.txt 此参数定义跟踪输出所要发送到的文件的名称（包括路径）。如果未提供路径，那么跟踪文件将写入 IBM Integration Bus 安装的整个机器的工作路径目录。在 Windows 上，缺省工作路径目录为 C:\ProgramData\Application Data\IBM\MQSI。在 Linux 上，缺省工作路径目录为 /var/mqsi。

所有模式参数均随其缺省值一起列出。 未指示缺省值时，表明缺省值是空字符串。
