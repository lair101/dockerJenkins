Parameter für das Muster HL7 an HL7 DFDL

Quelle
Musterparameter Standardeinstellung Beschreibung
Quellentyp Generisch Dieser Parameter definiert den Typ der Quellenanwendung. Der Parameter wird für die Auswahl des HL7DFDLInput-Knotens im Receiver-Nachrichtenfluss verwendet.
Quelleneinrichtung Dieser Parameter definiert die Quellenfunktion, die in Fehlernachrichten, Benachrichtigungen und standardmäßigen Journaling-Nachrichten integriert ist.
Quellen-ID Quellenanwendung Dieser Parameter definiert eine ID, mit der die Quelle in Fehlernachrichten, Benachrichtigungen und standardmäßigen Journaling-Nachrichten beschrieben wird.
Verbindungsdetails localhost:1111 Dieser Parameter definiert die TCP/IP-Verbindung für die Quellenanwendung und wird in der Form Hostname:Port angegeben.
Zeitlimit beim Warten auf Daten 60 Dieser Parameter definiert, wie lange der Knoten an einer Verbindung für weitere Daten empfangsbereit ist, nachdem das erste Datenbyte eingetroffen ist. Sie können einen beliebigen Zeitwert in Sekunden angeben. Der Standardwert beträgt 60 Sekunden. Sobald die festgelegte Zeitspanne überschritten ist, werden alle verfügbaren Daten an das Fehlerterminal gesendet.
Führendes MLLP-Byte 0B Dieser Parameter definiert das führende MLLP-Byte, das am Anfang des HL7-Datensatzes gelöscht werden muss.
Abschließende MLLP-Bytes 1C0D Dieser Parameter definiert die abschließenden MLLP-Bytes, die als HL7-Datensatzbegrenzer verwendet werden. Diese abschließenden MLLP-Bytes werden vom TCPIPServerInput-Knoten entfernt. Sie werden außerdem vom TCPIPClientOutput-Knoten hinzugefügt, der HL7-Bestätigungen zurückgibt.
Grenzwert für Folgefehler 0 Dieser Parameter definiert die maximale Anzahl an aufeinanderfolgenden Fehlern, die toleriert wird, bevor die Netzverbindung geschlossen wird. Jede nicht positive Zahl wird als unbegrenzter Wert behandelt und die Verbindung wird bei aufeinanderfolgenden Fehlern nicht beendet.

Die Verbindung wird immer beendet, wenn die Nachricht nicht das richtige führende MLLP-Byte enthält.

Anzahl der Ziele 1 Dieser Parameter definiert die Anzahl der Ziele, an die Quellennachrichten verteilt werden.
Teil 1 Dieser Parameter definiert, welcher Teil eines mehrteiligen Musters derzeit konfiguriert wird.

Zielparameter
Musterparameter Standardeinstellung Beschreibung
Zieltyp Generisch Dieser Parameter definiert den Typ der Zielanwendung. Der Parameter wird für die Auswahl des HL7DFDLOutput-Knotens im Nachrichtenfluss 'Sender' verwendet.
Kennung Ziel 1 Dieser Parameter definiert die Identität eines Ziels. Diese Identität wird in Fehlernachrichten, Berichtsnachrichten und Benachrichtigungen verwendet, die mit einem Ziel in Zusammenhang stehen.
Zieleinrichtung Dieser Parameter definiert die Einrichtung eines Ziels. Diese Einrichtung wird in Fehlernachrichten, Berichtsnachrichten und Benachrichtigungen verwendet, die mit einem Ziel in Zusammenhang stehen.
Reihenfolgeaktion Keine Wenn im Receiver-Nachrichtenfluss die Option für die Anwendung von Folgenummern ausgewählt ist, legt dieser Parameter fest, wie die Sequenzbildung für ein Ziel ausgeführt wird. Folgende Werte sind gültig:
  • Keine: Nachrichten werden in der Reihenfolge verarbeitet, in der sie eintreffen.
  • Strikt: Nachrichten können nicht in der falschen Reihenfolge verarbeitet werden. Wenn eine Nachricht fehlt, werden alle anderen Nachrichten zurückgehalten, bis die Nachricht eintrifft.
  • Nicht strikt: Nachrichten werden in der Regel der Reihe nach verarbeitet. Wenn eine Nachricht jedoch nicht innerhalb eines bestimmten Zeitraums eingetroffen ist, werden andere Nachrichten verarbeitet. Wenn die fehlende Nachricht nach dem Zeitlimit eintrifft, wird sie in den Datenstrom eingefügt.
Zeitlimit für Reihenfolge (Sekunden) Dieser Parameter definiert die Zeit in Sekunden, für die der Nachrichtenfluss der Nachrichten auf eine Nachricht wartet, die nicht in der richtigen Reihenfolge eintrifft. Nach Ablauf dieser Zeit wird der Nachrichtenfluss fortgesetzt. Wenn die fehlenden Nachrichten später eintreffen, werden sie in den Datenstrom eingefügt.
Nachrichtenfilter Nicht ausgewählt Dieser Parameter legt fest, ob die Nachrichten, die an ein Ziel gesendet werden, gefiltert werden. Wenn dieser Parameter ausgewählt ist, wird die Nachrichtenfiltertabelle angezeigt und es können Filterkriterien eingegeben werden.

Durch die Filterung ist sichergestellt, dass nur Nachrichten gesendet werden, deren Werte aus dem MSH-9 (Feld Message Type (Nachrichtentyp)) für Message Code (Nachrichtencode) und Trigger Event (Auslöserereignis) den Werten von Filter 1 und Filter 2 entsprechen.

Falls eine komplexere Filterung erforderlich ist, können Sie den ESQL-Code mit demselben Verfahren ändern.
Segmentfilter Nicht ausgewählt Dieser Parameter legt für ein Ziel fest, welche Segmente aus der Quellennachricht kopiert werden. Wenn dieser Parameter ausgewählt ist, wird die Segmentfiltertabelle angezeigt und es können Filterkriterien eingegeben werden.
Verbindungsdetails localhost:2222 Dieser Parameter definiert die TCP/IP-Verbindung für ein Ziel und wird in der Form Hostname:Port angegeben.
Standard-MLLP Ausgewählt Dieser Parameter legt fest, ob eine Standardkonfiguration für TCP/IP und MLLP verwendet wird. Die Standardkonfiguration weist folgende Werte auf:
  • Zeitlimit: 60 Sekunden
  • Führendes MLLP-Byte: 0B
  • Begrenzer: Nicht zutreffend
  • Wiederholungslimit: 3
  • Wiederholung protokollieren: true
  • Validierung: false

Wenn der Parameter nicht ausgewählt wird, wird die Parametergruppe für die Definition der vom Standard abweichenden Werte angezeigt.

Zielübertragungen
Musterparameter Standardeinstellung Beschreibung
Zeitlimit (Sekunden) 60 Dieser Parameter definiert in Sekunden das Zeitlimit für ein Ziel.
Führendes MLLP-Byte 0B Dieser Parameter definiert das führende MLLP-Byte, das den ausgehenden Datensätzen hinzugefügt wird.
Abschließende MLLP-Bytes 1C0D Dieser Parameter definiert den Begrenzer. Dieser Wert muss im hexadezimalen Format angegeben werden.

Wiederholung protokollieren Ausgewählt Dieser Parameter legt fest, ob jeder Versuch einer Nachrichtenzustellung protokolliert wird.
Prüfung Keine Dieser Parameter legt fest, ob die Nachricht anhand der HL7-Nachrichtengruppe validiert wird.
Optionen
Musterparameter Standardeinstellung Beschreibung
Quellenfeed Ausgewählt Dieser Parameter legt fest, ob eine Kopie der Quellennachricht generiert wird. Der Parameter Publish (Veröffentlichen) legt fest, ob der Quellenfeed in die Warteschlange Instanz.SRCF geschrieben oder im Thema Muster-ID/Receiver/Source veröffentlicht wird.
Folgenummern Nach Eingang Dieser Parameter legt fest, wie Nachrichtenfolgenummern für das Sequenzbildungsverhalten festgelegt werden, das durch den Parameter 'sequencing' (Sequenzbildung) für jedes Ziel definiert ist. Folgende Werte sind gültig:
  • Keine Reihenfolge
  • Nach Eingang: Folgenummern werden in der Reihenfolge zugeordnet, in der sie beim Knoten eintreffen.
  • Nach Inhalt: Folgenummern werden gemäß der Folgenummer zugeordnet, die im MSH-Header gefunden wird.
Bestätigung senden Ausgewählt Dieser Parameter definiert, ob die sendende Anwendung eine Standardantwort des Typs ACK oder NACK erwartet.
Journaling Standard Dieser Parameter legt die Optionen zur Journalführung fest. Folgende Werte sind gültig:
  • Keine: Keine Journalführung. Diese Option kann auch verwendet werden, wenn der Benutzer eines Musters einen Quellen-Feed verwenden möchte, um die Journalführung für seine eigenen Anforderungen auszuführen.
  • Standard: Es wird eine Kopie jeder Nachricht mit den vom Muster definierten standardmäßigen Kopfzeileninformationen erstellt. Diese Kopie wird im Thema Muster-ID/Receiver/Journal veröffentlicht oder in die Journalwarteschlange geschrieben (je nach Festlegung des Parameters Publish).
Kanonischer Feed Nicht ausgewählt Dieser Parameter legt fest, ob eine Nachrichtenkopie im kanonischen Format generiert wird. Ist der Parameter Publish ausgewählt, legt er fest, ob diese Nachrichtenkopie an die Warteschlange Muster-ID.CAN oder an das Thema Muster-ID/Canonical gesendet wird.
Restliche melden Ausgewählt Dieser Parameter legt fest, ob Remainder-Felder aufgelistet werden.

Wenn eine eingehende Nachricht Segmente enthält, die nicht in der HL7-Nachrichtengruppe modelliert sind, werden Remainder-Felder gesucht.

Ist dieser Parameter ausgewählt, legt der Parameter Publish (Veröffentlichen) fest, ob Nachrichten mit Remainder-Feldern an die Warteschlange Muster-ID.REM gesendet oder veröffentlicht werden.

Diese Aktion ist vor allem während der Entwicklung der Integration zwischen HL7-Anwendungen hilfreich.
Auf Duplikate überprüfen Nicht ausgewählt Dieser Parameter legt fest, ob doppelte Nachrichten gefunden werden, die aufgrund der eindeutigen Nachrichtenkennung im HL7-MSH-Header ermittelt werden.

Falls Duplikate gefunden werden, wird die folgende Aktion ausgeführt:

  • Wenn die Nachricht innerhalb eines Zeitlimitintervalls empfangen wird, wird sie nicht verarbeitet, aber dieselbe ACK-Meldung, die für die ursprüngliche Nachricht gesendet wurde, wird an den Sender zurückgegeben.
  • Nach Ablauf eines gewissen Zeitraums werden doppelte Nachrichten wie gewöhnlich verarbeitet.
Veröffentlichen Veröffentlichen Dieser Parameter legt fest, ob Journalnachrichten, Restnachrichten (Remainder) und sonstige Berichte veröffentlicht oder in Warteschlangen geschrieben werden.
Separate Sequenzwarteschlangen Nicht ausgewählt Dieser Parameter legt fest, ob zum Speichern von Nachrichten während der Sequenzbildung separate Warteschlangen für die einzelnen Musterinstanzen verwendet werden sollen. Standardmäßig befinden sich alle Nachrichten, die für die Sequenzbildung in einem Integrationsknoten gespeichert sind, zusammen in einer Systemwarteschlange.

Wenn dieser Parameter ausgewählt ist, wird ein konfigurierbarer Service für diese Musterinstanz definiert. Mit diesem konfigurierbaren Service werden die internen Warteschlangen definiert, die von den Resequence-Knoten in den Sender-Nachrichtenflüssen verwendet werden.
Verarbeitung von Duplikaten
Musterparameter Standardeinstellung Beschreibung
Duplikate melden Ausgewählt Dieser Parameter legt fest, ob doppelte Nachrichten, die innerhalb der angegebenen Zeit eintreffen, zur Generierung einer Fehlernachricht führen.
Ablaufzeit von ID-Nachrichten (Sekunden) 86400 Dieser Parameter definiert (in Sekunden), wie lange Kennungsnachrichten behalten werden, bevor sie ablaufen. Nach diesem Zeitraum werden Duplikate nicht erkannt, sondern wie gewöhnlich verarbeitet. Der Standardwert beträgt 24 Stunden.
Allgemein
Musterparameter Standardeinstellung Beschreibung
Beschreibung HL7-Muster Dieser Parameter definiert den Zweck des generierten Nachrichtenflusses und kann alle von Ihnen geforderten Einzelheiten enthalten.
Warteschlangenpräfix Dieser Parameter definiert eine Zeichenfolge, die Sie den Warteschlangennamen als Präfix voranstellen möchten. Die Standardwarteschlangen werden mit diesem Präfix und einer Standardgruppe von Suffixen erstellt.

Wenn dieser Parameter leer ist, wird der Name der Musterinstanz für das Entfernen von nicht alphanumerischen Zeichen verarbeitet und anschließend als Warteschlangenpräfix verwendet.
Nachrichtenflusspräfix Dieser Parameter definiert eine Zeichenfolge, die an den Anfang des Namens jedes Hauptnachrichtenflusses in einer generierten Musterinstanz hinzugefügt wird. Diese Zeichenfolge erleichtert die Ermittlung der Musterinstanz in Berichten und in der Überwachung.

Wenn dieser Parameter leer ist, wird der Name der Musterinstanz für das Entfernen von nicht alphanumerischen Zeichen verarbeitet und anschließend als Nachrichtenflusspräfix verwendet.
Scripts generieren Ausgewählt Dieser Parameter gibt an, ob ein MQSC-Implementierungsscript generiert wird.
Name der Tracedatei tracefile.txt Dieser Parameter definiert den Dateinamen einschließlich des Pfads, an den die Traceausgabe gesendet wird. Ist kein Pfad angegeben, wird die Tracedatei in das systemweite Arbeitspfadverzeichnis Ihrer IBM Integration Bus-Installation geschrieben. Unter Windows lautet das standardmäßige Arbeitspfadverzeichnis C:\ProgramData\Application Data\IBM\MQSI. Unter Linux ist das standardmäßige Arbeitspfadverzeichnis /var/mqsi.

Für alle Musterparameter sind die Standardwerte aufgeführt. Ist kein Standardwert angegeben, ist der Standardwert eine leere Zeichenfolge.
