Parameters for the Cross-Enterprise Document Sharing Consumer pattern

Find Documents
Pattern Parameter Default Description
Document Registry URL This parameter specifies the URL of your document registry (HTTP or HTTPS).
Input Directory This parameter specifies the directory for input files, in the form: 0bd58f55ed06424^^^&1.3.6.1.4.1.21367.2009.1.2.300&ISO
Output Directory This parameter specifies the directory to which output files from your query are written, in the form: 
<Document>
<uniqueId>1.42.20140903083332.2</uniqueId>
<repositoryUniqueId>1.19.6.24.109.42.1.5</repositoryUniqueId>
</Document>
Failure Output Directory This parameter specifies the directory to which failed queries are written.
Retrieve Documents
Pattern Parameter Default Description
Document Repository URL This parameter specifies the URL of your document repository (HTTP or HTTPS).
Input Directory This parameter specifies the directory for input files, in the form: 
<Document>
<uniqueId>1.42.20140903083332.2</uniqueId>
<repositoryUniqueId>1.19.6.24.109.42.1.5</repositoryUniqueId>
</Document>
Output Directory This parameter specifies the directory to which documents you have retrieved from the document query are written.
Failure Output Directory This parameter specifies the directory to which failed queries are written.

All the pattern parameters are listed with their default values. Where no default is indicated, the default value is an empty string.
