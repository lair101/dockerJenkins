Patrón Servicio sanitario: HIPAA a XML

Utilice el patrón Servicio sanitario: HIPAA a XML para convertir archivos HIPAA a XML.

Solución

El patrón Servicio sanitario: HIPAA a XML genera un flujo de mensajes que utiliza un modelo DFDL para analizar archivos de entrada HIPAA.

El flujo de mensajes lee los archivos de un directorio de entrada especificado. Cada archivo se analiza utilizando el modelo DFDL, se convierte a XML y se graba en un directorio de salida especificado.

Cada archivo de entrada debe ser un intercambio XML que contiene uno o más conjuntos de transacciones HIPAA. Se lee el archivo de entrada completo, se convierte a XML y se graba en el directorio de salida. Si hay errores de sintaxis en el archivo de entrada, éstos se detectan, pero el patrón no valida el contenido del archivo contrastándolo con la especificación HIPAA.

Después de generar una instancia del patrón Servicio sanitario: HIPAA a XML, puede modificar el flujo de mensajes generado. Por ejemplo, para leer otra fuente de entrada, personalizar las correlaciones o generar un formato de salida diferente.

Cuando se crea una instancia del patrón Servicio sanitario: HIPAA a XML, se crean una serie de bibliotecas con el nombre HIPAA_5010_conjunto_transacciones, donde conjunto_transacciones es el nombre del conjunto de transacciones HIPAA. Cada biblioteca contiene el modelo DFDL del conjunto de transacciones HIPAA específico; por ejemplo, la biblioteca HIPAA_5010_820A1 contiene el modelo DFDL del conjunto de transacciones HIPAA 820A1. Para mejorar la capacidad de respuesta de IBM Integration Toolkit al trabajar con la instancia de patrón elimine las referencias de biblioteca a conjuntos de transacciones que no desee utilizar. Para eliminar estas referencias, en la vista Desarrollo de aplicaciones, pulse con el botón derecho del ratón en la biblioteca HIPAAtoXMLMapping y seleccione Gestionar referencias de biblioteca. A continuación, desmarque las entradas de las bibliotecas que correspondan a los conjuntos de transacciones que no necesite.
