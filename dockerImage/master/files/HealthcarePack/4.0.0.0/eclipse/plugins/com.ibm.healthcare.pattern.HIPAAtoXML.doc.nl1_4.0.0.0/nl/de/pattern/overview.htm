Muster 'Healthcare: HIPAA in XML'

Verwenden Sie das Muster 'Healthcare: HIPAA in XML', um HIPAA-Dateien in XML zu konvertieren.

Lösung

Das Muster 'Healthcare: HIPAA in XML' generiert einen Nachrichtenfluss, der ein DFDL-Modell verwendet, um HIPAA-Eingabedateien zu parsen.

Der Nachrichtenfluss liest Dateien aus einem bestimmten Eingabeverzeichnis. Jede Datei wird anhand des DFDL-Modells geparst, in XML konvertiert und in ein bestimmtes Ausgabeverzeichnis geschrieben.

Jede Eingabedatei muss ein XML-Interchange sein, der mindestens eine HIPAA-Transaktionsgruppe enthält. Die gesamte Eingabedatei wird gelesen, in XML konvertiert und in das Ausgabeverzeichnis geschrieben. Syntaxfehler in der Eingabedatei werden erkannt, aber das Muster prüft nicht, ob der Inhalt der Datei der HIPAA-Spezifikation entspricht.

Nachdem Sie eine Instanz des Musters 'Healthcare: HIPAA in XML' generiert haben, können Sie den generierten Nachrichtenfluss modifizieren, um beispielsweise aus einer anderen Eingabequelle zu lesen, die Zuordnungen anzupassen oder ein anderes Ausgabeformat zu generieren.

Wenn Sie eine Instanz des Musters 'Healthcare: HIPAA in XML' erstellen, werden mehrere Bibliotheken mit dem Namen HIPAA_5010_Transaktionsgruppe erstellt, wobei Transaktionsgruppe für den Namen der Transaktionsgruppe steht. Jede Bibliothek enthält das DFDL-Modell für die angegebene HIPAA-Transaktionsgruppe. Die Bibliothek HIPAA_5010_820A1 enthält beispielsweise das DFDL-Modell für die HIPAA-Transaktionsgruppe 820A1. Zur Verbesserung der Reaktionsfähigkeit von IBM Integration Toolkit bei der Arbeit mit der Musterinstanz entfernen Sie die Bibliotheksreferenzen für Transaktionsgruppen, die Sie nicht verwenden möchten. Zum Entfernen dieser Referenzen klicken Sie in der Ansicht Anwendungsentwicklung mit der rechten Maustaste auf die Bibliothek HIPAAtoXMLMapping und klicken Sie dann auf Bibliotheksreferenzen verwalten. Wählen Sie anschließend die Einträge für Bibliotheken ab, die den nicht mehr benötigten Transaktionsgruppen entsprechen.
