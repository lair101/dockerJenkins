“Healthcare：从 HIPAA 到 XML”模式

使用“Healthcare：从 HIPAA 到 XML”模式可将 HIPAA 文件转换成 XML。

解决方案

“Healthcare：从 HIPAA 到 XML”模式生成使用 DFDL 模型来解析 HIPAA 输入文件的消息流。

该消息流从指定的输入目录读取文件。每个文件均使用 DFDL 模型进行解析，然后转换成 XML，并写入指定的输出目录。

每个输入文件必须是包含一个或多个 HIPAA 事务集的 XML 交换。读取整个输入文件后，会将其转换成 XML，并写入输出目录。输入文件中的语法错误会被检测到，但模式不会根据 HIPAA 规范来验证文件的内容。

生成“Healthcare：从 HIPAA 到 XML”模式的实例后，可以修改生成的消息流。例如，要从其他输入源进行读取，请定制映射，或者生成其他输出格式。

在创建“Healthcare：从 HIPAA 到 XML”模式的实例后，将创建名为 HIPAA_5010_transaction_set 的大量库，其中，transaction_set 是 HIPAA 事务集的名称。每个库都包含指定的 HIPAA 事务集的 DFDL 模型；例如，HIPAA_5010_820A1 库包含 HIPAA 事务集 820A1 的 DFDL 模型。要提高处理模式实例时 IBM Integration Toolkit 的响应性，请除去不想使用的事务集的库引用。要除去这些引用，请在应用程序开发视图中，右键单击 HIPAAtoXMLMapping 库，然后单击“管理库引用”。然后，取消选择对应于不需要的事务集的库的条目。
