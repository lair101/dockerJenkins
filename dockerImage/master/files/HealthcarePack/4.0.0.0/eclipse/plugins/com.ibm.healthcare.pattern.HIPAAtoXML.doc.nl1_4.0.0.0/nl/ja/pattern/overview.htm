「Healthcare: HIPAA to XML」パターン

「Healthcare: HIPAA to XML」パターンは、HIPAA ファイルを XML に変換するのに使用します。

ソリューション

「Healthcare: HIPAA to XML」パターンは、HIPAA 入力ファイルを解析するために DFDL モデルを使用するメッセージ・フローを生成します。

そのメッセージ・フローは指定された入力ディレクトリーからファイルを読み取ります。各ファイルは DFDL モデルを使用して解析され、XML に変換されて、指定の出力ディレクトリーに書き出されます。

各入力ファイルは、1 つ以上の HIPAA トランザクション集合を含む XML 交換でなければなりません。入力ファイル全体が読み取られ、XML に変換されて、出力ディレクトリーに書き出されます。入力ファイル内の構文エラーが検出されますが、このパターンは HIPAA 仕様に照らしてファイルの内容を検証することは行いません。

「Healthcare: HIPAA to XML」パターンのインスタンスを生成した後、生成されたメッセージ・フローを変更することができます。例えば、別の入力ソースから読み取りを行うには、マッピングをカスタマイズするか、別の出力フォーマットを生成します。

「Healthcare: HIPAA to XML」パターンのインスタンスを生成すると、いくつかのライブラリーが、 HIPAA_5010_transaction_set という名前で作成されます。ここで transaction_set は、HIPAA トランザクション集合の名前です。各ライブラリーには、指定された HIPAA トランザクション集合の DFDL モデルが含まれます。例えば、HIPAA_5010_820A1 ライブラリーには、HIPAA トランザクション集合 820A1 の DFDL モデルが含まれます。パターン・インスタンスを処理する際に IBM Integration Toolkit の即応性を向上させるには、使用したくないトランザクション集合のライブラリー参照を削除します。そのような参照を削除するには、 「アプリケーション開発」ビューで、HIPAAtoXMLMapping ライブラリーを右クリックし、 「ライブラリー参照の管理 (Manage Library References)」をクリックします。次に、必要ないトランザクション集合に対応するライブラリーの項目を選択解除します。
