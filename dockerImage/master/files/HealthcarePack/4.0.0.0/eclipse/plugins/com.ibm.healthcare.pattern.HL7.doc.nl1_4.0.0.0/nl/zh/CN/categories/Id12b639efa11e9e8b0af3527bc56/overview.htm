Healthcare

IBM Integration Bus Healthcare Pack 基于 IBM Integration Bus 构建，以便为 Healthcare 中的应用程序提供支持。

“Healthcare：从 HIPAA 到 XML”模式

“Healthcare：从 HIPAA 到 XML”模式生成可用于将 HIPAA 文件转换成 XML 文件的消息流。

“Healthcare：HL7 变换”模式

“Healthcare：HL7 变换”模式生成可用于组合 HL7 消息的图形数据图。

“Healthcare：从 HL7 到 HL7 DFDL”模式

“Healthcare：从 HL7 到 HL7 DFDL”模式将能够发送 HL7 V2 消息的应用程序与一个或多个能够接收 HL7 消息的应用程序集成。应用程序必须能够使用基于 TCP/IP 的 MLLP 来发送和接收消息。 此模式使用 DFDL 消息模型。如果要使用 MRM 和 HL7v25P 消息集，而不使用 DFDL，那么可使用提供的此模式的替代版本，名为“Healthcare：从 HL7 到 HL7”模式。

“Healthcare：从 HL7 到报告”模式

“Healthcare：从 HL7 到报告”模式将能够发送 HL7 V2 消息的应用程序与文本报告生成集成。文本报告通过使用 PHP 脚本进行编写，并且对 HL7 消息数据具有完全访问权。已完成的报告将定期交付到目标文件系统目录，并可以选择进一步复制到外部 FTP/SFTP 站点。

“Healthcare：医疗设备到 EMR”模式

“Healthcare：医疗设备到 EMR”模式将医疗设备与能够接收 Health Level Seven International (HL7) V2 观察结果消息 (ORU R01) 的电子医疗记录 (EMR) 应用程序集成。此应用程序必须能够使用基于 TCP/IP 的 Minimal Lower Layer Protocol (MLLP) 协议接收 HL7 ORU R01 消息。

“Healthcare：Web 服务到 DICOM”模式

使用“Healthcare：Web 服务到 DICOM”模式可以将使用 Web 服务编写的应用程序与支持 C-FIND 和 C-MOVE 操作的 DICOM 应用程序集成。借助 IBM Integration Bus 实现的 Web 服务，可以使用此模式来查询 DICOM 归档 (PACS) 中的患者、研究、系列和图像。

“Healthcare：Home Health”模式

使用“Healthcare：Home Health”模式以使您的医疗系统能够接受来自 HL7 V2.6 兼容设备的 Web 服务请求。这允许随后将 Home Health 设备收集并发送到应用程序托管设备 (AHD) 的数据发送到请求的临床应用程序。

Healthcare：患者标识交叉引用管理器

使用“Healthcare：患者标识交叉引用管理器”模式可创建与 IHE PIX 概要文件一致的“患者标识交叉引用管理器”。
