Healthcare

Das IBM Integration Bus Healthcare-Paket basiert auf IBM Integration Bus und unterstützt Anwendungen im Gesundheitswesen.

Muster 'Healthcare: HIPAA in XML'

Das Muster Healthcare: HIPAA in XML generiert einen Nachrichtenfluss, den Sie zum Transformieren von HIPAA-Dateien in XML-Dateien verwenden können.

Muster 'Healthcare: HL7-Transformation'

Das Muster Healthcare: HL7-Transformation generiert grafische Datenzuordnungen, mit denen Sie HL7-Nachrichten assemblieren können.

Muster 'Healthcare: HL7 an HL7 DFDL'

Das Muster Healthcare: HL7 an HL7 DFDL integriert eine Anwendung, die HL7-Nachrichten der Version 2 senden kann, in mindestens eine andere Anwendung, die HL7-Nachrichten empfangen kann. Die Anwendungen müssen in der Lage sein, die Nachrichten unter Verwendung von MLLP über TCP/IP zu senden und zu empfangen. Dieses Muster verwendet das DFDL-Nachrichtenmodell. Eine alternative Version dieses Musters mit dem Namen Healthcare: HL7 an HL7 wird für den Fall bereitgestellt, dass Sie MRM und die HL7v25P-Nachrichtengruppe anstelle von DFDL verwenden möchten.

Muster 'Healthcare: HL7 an Berichte'

Das Muster Healthcare: HL7 an Berichte integriert eine Anwendung, die HL7-Nachrichten der Version 2 mit Textberichtgenerierung senden können. Textberichte werden mithilfe von PHP-Scripts geschrieben und verfügen über einen uneingeschränkten Zugriff auf die HL7-Nachrichtendaten. Vollständige Berichte werden in regelmäßigen Abständen an ein Verzeichnis auf dem Zieldateisystem übermittelt. Dabei besteht auch die Möglichkeit, die Berichte zu kopieren und auf einer externen FTP/SFTP-Site zur Verfügung zu stellen.

Muster 'Healthcare: Medizinische Geräte an EKA'

Das Muster Healthcare: Medizinische Geräte an EKA integriert medizinische Geräte in eine Anwendung für elektronische Patientenakten (EKA), die Untersuchungsergebnisnachrichten (ORU R01) des Standards HL7 v2 (Health Level Seven International) empfangen kann. Die Anwendung muss in der Lage sein, HL7-ORU-R01-Nachrichten unter Verwendung des MLLP-Protokolls (MLLP = Minimal Lower Layer Protocol) über TCP/IP zu empfangen.

Muster 'Healthcare: Web-Service an DICOM'

Das Muster Healthcare: Web-Service an DICOM integriert eine Anwendung, die mithilfe von Web-Services geschrieben, wurde, in DICOM-Anwendungen, die C-FIND- und C-MOVE-Operationen unterstützen. Mit diesem Muster können mithilfe eines über IBM Integration Bus implementierten Web-Service Patienten, Untersuchungen, Serien und Bilder aus einem DICOM-Archiv (PACS) abgefragt werden.

Muster 'Healthcare: Ambulante Pflege'

Mithilfe des Musters Healthcare: Ambulante Pflege ermöglichen Sie Ihren Gesundheitskontrollsystemen, Web-Service-Anforderungen von Geräten zu akzeptieren, die mit HL7 v2.6 kompatibel sind. Dadurch können Daten, die von Geräten für die ambulante Pflege erfasst und an das Anwendungshostgerät gesendet werden, anschließend an die anfordernde klinische Anwendung gesendet werden.

Muster 'Healthcare: Patient Identifier Cross-reference Manager'

Verwenden Sie das Muster Healthcare: Patient Identifier Cross-reference Manager, um einen Patient Identifier Cross-reference Manager zu erstellen, der dem IHE-PIX-Profil entspricht.
