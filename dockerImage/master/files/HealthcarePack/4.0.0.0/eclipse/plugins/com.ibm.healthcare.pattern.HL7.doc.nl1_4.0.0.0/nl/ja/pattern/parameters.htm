「HL7 to HL7」パターンのパラメーター

ソース
パターン・パラメーター デフォルト 説明
ソース・タイプ Generic このパラメーターは、ソース・アプリケーションのタイプを定義します。 このパラメーターは、受信側フローで GenericHL7Input ノードを選択するために使用します。
ソース・ファシリティー このパラメーターは、エラー・メッセージ、通知、および標準のジャーナル・メッセージに含まれるソース・ファシリティーを定義します。
ソース ID ソース・アプリケーション このパラメーターは、エラー・メッセージ、通知、および標準のジャーナル・メッセージ内でのソースの記述に使用される ID を定義します。
接続の詳細 localhost:1111 このパラメーターは、ソース・アプリケーションの TCP/IP 接続を hostname:port の形式で定義します。
データ待機のタイムアウト (秒) 60 このパラメーターは、データの先頭バイトが到着した後、ノードが接続上で後続のデータを listen する時間の長さを定義します。 秒単位で任意の時間の長さを指定できます。 デフォルトは 60 秒です。 指定時間を超過すると、使用可能なデータがすべて Failure ターミナルに送信されます。
先頭の MLLP バイト 0B このパラメーターは、HL7 レコードの先頭から切り取る必要のある先行 MLLP バイトを定義します。
末尾の MLLP バイト 1C0D このパラメーターは、HL7 レコード区切り文字として使用される末尾の MLLP バイトを定義します。 この末尾の MLLP バイトは、TCPIPServerInput ノードによって削除されます。 また、これらのバイトは、HL7 肯定応答を返す TCPIPClientOutput ノードによって追加されます。
連続エラーの限度 0 このパラメーターは、ネットワーク接続を閉じる前に許容される連続エラーの最大数を定義します。 正でない数値は無限大として処理され、連続する障害があっても接続は終了しません。

メッセージに正しい先頭の MLLP バイトが含まれていない場合、接続は常に終了されます。

宛先
パターン・パラメーター デフォルト 説明
宛先の数 1 このパラメーターは、ソース・メッセージを配布する宛先の数を定義します。
パート 1 このパラメーターは、マルチパート・パターンのどのパートを現在構成しているかを定義します。

マルチパート・パターン・インスタンスの構成について詳しくは、パターンの実行を参照してください。

宛先パラメーター
パターン・パラメーター デフォルト 説明
宛先タイプ Generic このパラメーターは、宛先アプリケーションのタイプを定義します。 このパラメーターは、送信側フローで GenericHL7Output ノードを選択するために使用します。
ID 宛先 1 このパラメーターは、宛先の ID を定義します。 この ID は、宛先に関連するエラー、レポート、および通知の各メッセージで使用されます。
宛先ファシリティー このパラメーターは、宛先のファシリティーを定義します。 このファシリティーは、宛先に関連するエラー、レポート、および通知の各メッセージで使用されます。
順序付けアクション なし シーケンス番号を適用するオプションが受信側フローで選択される場合、このパラメーターは、順序付けを宛先に適用する方法を決定します。 有効な値は以下のとおりです。
  • なし: メッセージは到着した順に処理されます。
  • 厳密: メッセージは必ず順番どおりに処理されます。 1 つのメッセージが欠落していると、そのメッセージが到着するまで、他のすべてのメッセージは保留されます。
  • 緩い: 全般的に、メッセージは順番に処理されます。 ただし、1 つのメッセージが指定された期間内に到着しないと、他のメッセージが処理されます。 欠落していたメッセージがタイムアウトの後に到着すると、欠落していたメッセージがストリームに挿入されます。
順序付けタイムアウト (秒) このパラメーターは、正しい順序で到着しないメッセージをメッセージのフローが待機する時間 (秒) を定義します。 この時間を経過した後、メッセージはフローを続行します。 欠落していたメッセージが後から到着すると、欠落していたメッセージはストリームに挿入されます。
メッセージ・フィルタリング クリア このパラメーターは、宛先に送信されるメッセージをフィルターに掛けるかどうかを決定します。 このパラメーターを選択すると、メッセージ・フィルタリング・テーブルが表示され、フィルタリング条件を入力できます。

フィルタリングにより、MSH-9 (「メッセージ・タイプ」フィールド) の「メッセージ・コード」および「トリガー・イベント」からの値が「フィルター 1」および「フィルター 2」の値に一致するメッセージのみが残ります。

さらに複雑なフィルター処理が必要な場合は、同様の方法で ESQL コードを変更してください。

セグメント・フィルタリング クリア このパラメーターは、どのセグメントがソース・メッセージから宛先にコピーされるかを決定します。 このパラメーターを選択すると、セグメント・フィルタリング・テーブルが表示され、フィルタリング条件を入力できます。
接続の詳細 localhost:2222 このパラメーターは、宛先用の TCP/IP 接続を hostname:port 形式で定義します。
標準 MLLP 選択 このパラメーターは、TCP/IP および MLLP についてデフォルトの構成を使用するかどうかを決定します。 デフォルト構成の値は以下のとおりです。
  • タイムアウト: 60 秒
  • 先頭の MLLP バイト: 0B
  • 区切り文字: 適用外
  • 再試行限界: 3
  • 再試行をログに記録: true
  • 検証: false

このパラメーターの選択を解除すると、標準以外の値を定義するためのパラメーター・グループが表示されます。

セグメント はい   このパラメーターは、セグメントをフィルターに掛けるために使用するセグメント・コードを定義します。 3 文字のセグメント・コードがセグメント・フィルター・テーブルにあるいずれかの項目に一致すると、そのセグメントは出力メッセージから除外されます。
宛先通信
パターン・パラメーター デフォルト 説明
タイムアウト (秒) 60 このパラメーターは、宛先のタイムアウトを秒単位で定義します。
先頭の MLLP バイト 0B このパラメーターは、出力レコードに追加される、先頭の MLLP バイトを定義します。
末尾の MLLP バイト 1C0D このパラメーターは、区切り文字が何であるかを定義します。 この値は 16 進数で指定する必要があります。
再試行限界 3 このパラメーターは、送信側フローが宛先アプリケーションへの HL7 メッセージの送信を試行する回数を定義します。
再試行をログに記録 選択 このパラメーターは、メッセージの送達試行をそれぞれログに記録するかどうかを決定します。
検証 なし このパラメーターは、HL7 メッセージ・セットに照らしてメッセージを検証するかどうかを指定します。
オプション
パターン・パラメーター デフォルト 説明
ソース・フィード 選択 このパラメーターは、ソース・メッセージのコピーを生成するかどうかを決定します。 「パブリッシュ」パラメーターは、ソース・フィードをキュー instance.SRCF に書き込むか、トピック PatternId/Receiver/Source にパブリッシュするかを決定します。
シーケンス番号 到着ベース このパラメーターは、各宛先の順序付けパラメーターによって定義される順序付けの動作に対してメッセージのシーケンス番号をどのように設定するかを決定します。 有効な値は以下のとおりです。
  • 順序付けなし
  • 到着ベース: ノードに到着した順にシーケンス番号を割り当てます。
  • 内容ベース: MSH ヘッダー内に見つかるシーケンス番号からシーケンス番号を割り当てます。
肯定応答の送信 選択 このパラメーターは、送信側アプリケーションが標準の ACK または NACK 応答を想定するかどうかを決定します。
ジャーナリング 標準 このパラメーターは、ジャーナリングのオプションを決定します。有効な値は以下のとおりです。
  • なし: ジャーナリングはありません。 このオプションは、パターンのユーザーが、自身の要件に対するジャーナリングの実行のためにソース・フィードを使用する場合に使用することもできます。
  • 標準: 各メッセージのコピーが、パターンで定義された標準ヘッダー情報とともに生成されます。 このコピーは、「パブリッシュ」パラメーターの指定に従って、トピック PatternId/Receiver/Journal にパブリッシュされるか、ジャーナル・キューに書き込まれます。
正規のフィード クリア このパラメーターは、正規の形式でメッセージのコピーが生成されるかどうかを指定します。 このパラメーターを選択した場合は、「パブリッシュ」パラメーターにより、メッセージのこのコピーが PatternId.CAN キューに送信されるか、トピック PatternId/Canonical に送信されるかが決まります。
剰余の報告 選択 このパラメーターは、剰余フィールドを報告するかどうかを決定します。

HL7 メッセージ・セットでモデル化されていないセグメントが着信メッセージに含まれていると、剰余フィールドが見つかります。

このパラメーターを選択した場合は、「パブリッシュ」パラメーターにより、剰余フィールドを伴うメッセージが PatternId.REM キューに送信されるか、パブリッシュされるかが決まります。

このアクションは、HL7 アプリケーションとの統合を開発している期間に、特に便利に使用できます。

重複をチェック 選択 このパラメーターは、重複したメッセージを検出するかどうかを決定します。 重複は、HL7 MSH ヘッダーにある固有のメッセージ ID によって判断されます。

重複が検出されると、以下のアクションが実行されます。

  • メッセージがタイムアウト期間内に受信された場合、そのメッセージは処理されませんが、元のメッセージに対して送信されたのと同じ ACK が送信側に返されます。
  • タイムアウト期間が切れた後は、重複したメッセージがあっても、通常どおりに処理されます。
パブリッシュ パブリッシュ このパラメーターは、ジャーナル・メッセージ、剰余メッセージ、およびその他のレポートが、パブリッシュされるか、キューに書き込まれるかを決定します。
個別のシーケンス・キュー クリア このパラメーターは、順序付けの実行中にメッセージを格納する各パターン・インスタンスに対して別々のキューを使用するかどうかを決定します。 デフォルトでは、順序付けのために Integration ノード上に格納されるすべてのメッセージは、1 つのシステム・キューに保持されます。

このパラメーターを選択すると、このパターン・インスタンスに対して構成可能サービスが定義されます。 この構成可能サービスは、送信側フローの Resequence ノードで使用される内部キューを定義するために使用します。

重複処理
パターン・パラメーター デフォルト 説明
重複を報告 選択 このパラメーターは、指定した時間内に重複したメッセージが到着した場合にエラー通知メッセージを生成するかどうかを決定します。
ID メッセージの期限 (秒) 86400 このパラメーターは、期限が切れる前に ID メッセージが保持される時間 (秒単位) を定義します。 この期限が経過した後は、重複は認識されず、通常どおりに処理されます。 デフォルトは 24 時間です。
その他
パターン・パラメーター デフォルト 説明
説明 HL7 パターン このパラメーターは、生成されるフローの目的と必要に応じた任意の詳細を定義します。
キュー接頭部 このパラメーターは、キュー名の接頭部のストリングを定義します。標準キューは、この接頭部と一連の標準の接尾部付きで作成されます。

このパラメーターをブランクのままにすると、パターン・インスタンス名から非英数字を削除する処理が行われ、その処理されたパターン・インスタンス名がキューの接頭部として使用されます。
フロー接頭部 このパラメーターは、生成されるパターン・インスタンスの各メイン・メッセージ・フロー名の先頭に追加するストリングを定義します。このストリングは、レポート内およびモニターにおいて、パターン・インスタンスのコンポーネントを識別するのに役立ちます。

このパラメーターをブランクのままにすると、パターン・インスタンス名から非英数字を削除する処理が行われ、その処理されたパターン・インスタンス名がフローの接頭部として使用されます。
スクリプトの生成 選択 このパラメーターは、MQSC デプロイメント・スクリプトを生成するかどうかを決定します。
トレース・ファイル名 tracefile.txt このパラメーターは、トレース出力の送信先であるファイルの名前 (パスを含む) を定義します。パスを指定しない場合、トレース・ファイルは、IBM Integration Bus のインストール済み環境のマシン全体の作業パス・ディレクトリーに書き込まれます。Windows でのデフォルトの作業パス・ディレクトリーは C:¥ProgramData¥Application Data¥IBM¥MQSI です。Linux でのデフォルトの作業パス・ディレクトリーは /var/mqsi です。

すべてのパターン・パラメーターとそのデフォルト値をリストします。 デフォルトが示されていない場合、デフォルト値は空ストリングです。
