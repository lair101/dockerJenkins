Muster 'Healthcare: HL7 an HL7'

Das Muster Healthcare: HL7 an HL7 integriert eine Anwendung, die HL7-v2-Nachrichten (Health Level Seven International) senden kann, in eine oder mehrere Anwendungen, die HL7-Nachrichten empfangen können. Die Anwendungen müssen in der Lage sein, HL7-Nachrichten unter Verwendung des MLLP-Protokolls (MLLP = Minimal Lower Layer Protocol) über TCP/IP zu senden und zu empfangen.

Hinweis: Das Muster 'Healthcare: HL7 an HL7' steht in IBM WebSphere Message Broker Connectivity Pack for Healthcare Version 7.0 zur Verfügung. Es ist auch in der vorliegenden Version von IBM Integration Bus Healthcare Pack verfügbar und wird dort noch unterstützt. Es wird dennoch empfohlen, stattdessen möglichst das Muster 'Healthcare: HL7 an HL7 DFDL' zu verwenden. Dieses Muster verwendet das DFDL-Nachrichtenmodell anstelle von MRM und der HL7v25P-Nachrichtengruppe. Das DFDL-Nachrichtenmodell bietet folgende Vorteile:

  • DFDL ist ein Format, das auf offenen Standards basiert, während MRM und die HL7v25P-Nachrichtengruppe zu IBM Integration Bus gehören.
  • Im Vergleich zu MRM und der HL7v25P-Nachrichtengruppe stellt der DFDL-Editor einfachere Tools für die Entwicklung und den Test von Erweiterungen für das HL7-Schema bereit.
  • Das DFDL-Nachrichtenmodell unterstützt HL7 Version 2.7, 2.6, 2.5.1 und ältere Versionen, während MRM und die HL7v25P-Nachrichtengruppe nur HL7 Version 2.5.1 und ältere Versionen unterstützen.

Wichtig:

Sie müssen die HL7-Nachrichtengruppe vor der Implementierung einer Musterinstanz im Integrationsknoten implementieren. Mit der Nachrichtengruppe werden HL7-Nachrichten bei ihrer Verarbeitung durch die Nachrichtenflüsse syntaktisch analysiert. Die vom Muster generierten Nachrichtenflüsse können erst nach der Implementierung der Nachrichtengruppe im Integrationsknoten implementiert werden.

Sie können die HL7-Nachrichtengruppe aus der Seite Ressourcen für das Muster in Ihren Arbeitsbereich importieren.

Da die Erstellung der HL7-Nachrichtengruppe lange dauert und Ihren Arbeitsbereich erheblich verlangsamen kann, wird die folgende Vorgehensweise empfohlen:

  • Importieren Sie die HL7-Nachrichtengruppe in Ihren Arbeitsbereich.
  • Fügen Sie die HL7-Nachrichtengruppe einer BAR-Datei hinzu.
  • Implementieren Sie die BAR-Datei im Integrationsknoten.
  • Löschen Sie die HL7-Nachrichtengruppe und die BAR-Datei aus Ihrem Arbeitsbereich.

Wenn Sie den grafischen Mapper für Nachrichtentransformationen verwenden möchten, müssen Sie die HL7-Nachrichtengruppe in Ihrem Arbeitsbereich behalten. Sie können die Erstellungszeit für die Nachrichtengruppe verkürzen, indem Sie die HL7-Kapitel aus der Nachrichtengruppe löschen. Die Nachrichtendefinitionsdateien segments.mxsd , fields.mxsd und datatypes.mxsd müssen in der Nachrichtengruppe verbleiben, da sie vom Muster benötigt werden.

Optionen

Der Receiver-Nachrichtenfluss im Muster 'Healthcare: HL7 an HL7' unterstützt die folgenden Optionen:

  • Empfang von HL7-Nachrichten unter Verwendung von MLLP über TCP/IP
  • Validierung und Parsing der Nachrichten
  • Überwachung eingehender Nachrichten
  • Überprüfung und Verarbeitung doppelter Nachrichten
  • Unterstützung von Sequenzbildungsoptionen
  • Veröffentlichen oder Senden an eine Warteschlange (jede eingehende Nachricht für die Verarbeitung außerhalb des Musters)
  • Senden der Ausgabe an eine Warteschlange zur Transformation und Weiterleitung
  • Behandlung von Ausnahmebedingungen

Der Nachrichtenfluss 'TransformAndRoute' im Muster 'Healthcare: HL7 an HL7' unterstützt die folgenden Optionen:

  • Transformation in ein kanonisches XML-Format
  • Veröffentlichen oder Senden an eine Warteschlange (die kanonische eingehende Nachricht für die Verarbeitung außerhalb des Musters)
  • Auflisten zusätzlicher Felder, die in HL7-Segmenten gefunden werden
  • Verteilung von Nachrichten an mehrere, gesondert konfigurierbare Ziele
  • Filtern der an die einzelnen Ziele zu sendenden Nachrichten
  • Filtern der Segmente, die für jedes Ziel einbezogen werden sollen
  • Anpassungspunkte, damit zielspezifische Änderungen einbezogen werden können, die vom Standard abweichen
  • Transformation in HL7 und Senden der Ausgabe an die Warteschlange für den Nachrichtenfluss 'Sender' für jedes Ziel
  • Behandlung von Ausnahmebedingungen

Die einzelnen Sender-Nachrichtenflüsse im Muster 'Healthcare: HL7 an HL7' unterstützen die folgenden Optionen:

  • Lesen transformierter Nachrichten für dieses Ziel
  • Validierung transformierter Nachrichten
  • Sicherstellen, dass Nachrichten in Übereinstimmung mit den Sequenzbildungsoptionen der Reihe nach zugestellt werden
  • Benachrichtigung, wenn Nachrichten in falscher Reihenfolge eintreffen
  • Zustellung von HL7-Nachrichten an die Zielanwendung unter Verwendung von MLLP über TCP/IP
  • Behandlung von Ausnahmebedingungen
