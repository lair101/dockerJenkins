Healthcare

IBM Integration Bus Healthcare Pack se basa en IBM Integration Bus para proporcionar soporte para aplicaciones en el área sanitaria.

Patrón Servicio sanitario: HIPAA a XML

El patrón Servicio sanitario: HIPAA a XML genera un flujo de mensajes que se puede utilizar para transformar archivos HIPAA en archivos XML.

Patrón Servicio sanitario: Transformación de HL7

El patrón Servicio sanitario: Transformación de HL7 genera correlaciones de datos gráficos que se pueden utilizar para ensamblar mensajes de HL7.

Patrón Servicio sanitario: HL7 a DFDL HL7

El patrón Servicio sanitario: HL7 a DFDL HL7 integra una aplicación que puede enviar mensajes HL7 v2 con una o más aplicaciones que pueden recibir mensajes HL7. Las aplicaciones deben poder enviar y recibir los mensajes utilizando MLLP a través de TCP/IP. Este patrón utiliza el modelo de mensaje DFDL. Se proporciona una versión alternativa de este patrón, denominado Servicio sanitario: HL7 a HL7, en caso de que desee utilizar MRM y el conjunto de mensajes HL7v25P en lugar de DFDL.

Patrón Servicio sanitario: HL7 a informes

El patrón Servicio sanitario: HL7 a informes integra una aplicación que puede enviar mensajes HL7 v2 con la generación de informes de texto. Los informes de texto se escriben utilizando scripts PHP y tienen acceso completo a los datos de mensajes HL7. Los informes completados se entregan periódicamente a un directorio de destino del sistema de archivos con una opción para copiar los informes a un sitio FTP/SFTP externo.

Patrón Servicio sanitario: Dispositivos médicos a EMR

El patrón Servicio sanitario: Dispositivos médicos a EMR integra dispositivos médicos con una aplicación de Historial clínico electrónico (HCE) que pueda recibir mensajes de resultados de observación (ORU R01) de HL7 (Health Level Seven International) v2. La aplicación debe poder recibir mensajes HL7 utilizando el protocolo MLLP (Minimal Lower Layer Protocol) sobre TCP/IP.

Patrón Servicio sanitario: Servicio web a DICOM

Puede utilizar el patrón Servicio sanitario: Servicio web a DICOM para integrar una aplicación creada utilizando servicios web con aplicaciones DICOM que admiten operaciones C-FIND y C-MOVE. Puede utilizar el patrón para consultar pacientes, estudios, series e imágenes de un archivo DICOM (PACS) utilizando un servicio web implementado por IBM Integration Bus.

Patrón Servicio sanitario: Salud en el hogar

Utilice el patrón Servicio sanitario: Salud en el hogar para permitir que sus sistemas de salud acepten solicitudes de servicio web procedentes de dispositivos compatibles con HL7 v2.6. Esto permite que los datos que los dispositivos de Salud en el hogar recopilan y envían al dispositivo de alojamiento de aplicaciones (AHD) se envíen a la aplicación clínica solicitante.

Servicio sanitario: Patient Identifier Cross-reference Manager

Utilice el patrón Servicio sanitario: Patient Identifier Cross-reference Manager para crear un Patient Identifier Cross-reference Manager que se ajuste al perfil PIX de IHE.
