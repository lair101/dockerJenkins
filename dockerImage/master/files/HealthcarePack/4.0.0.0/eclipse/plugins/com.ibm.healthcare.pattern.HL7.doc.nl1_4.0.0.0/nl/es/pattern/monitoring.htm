Supervisión del patrón Servicio sanitario: HL7 a HL7

Los flujos que genera el patrón Servicio sanitario: HL7 a HL7 se supervisan mediante la supervisión basada en la web. En la vista de supervisión se muestran estadísticas TCP/IP que publican los flujos de mensajes. Las estadísticas se aumentan con información almacenada en propiedades definidas por el usuario de los flujos de mensajes. En este tema se explica cómo se denominan estas propiedades definidas por el usuario y la información que contienen.

Si utiliza nodos HL7 fuera del patrón para crear integraciones personalizadas, puede integrar los flujos de mensajes con la vista de supervisión configurando las mismas propiedades definidas por el usuario.

Propiedades definidas por el usuario

El patrón Servicio sanitario: HL7 a HL7 siempre tiene un solo flujo Receiver y uno o más destinos. Por lo tanto, El identificador y el recurso de aplicación de origen están fijados para todos los destinos. El identificador y el recurso de aplicación de destino variarán según el flujo de mensajes de destino que inspeccione.

Nombre de la propiedad Valor Ejemplos
Identifier1 El nombre de la instancia de patrón ADT
Identifier2 Una serie literal según si es un flujo de mensajes Receiver o Sender Origen o Destino
Identifier3 El identificador HL7 de la aplicación de origen o destino según si es un flujo de mensajes Receiver o Sender Registro de paciente
Identifier4 El recurso HL7 de la aplicación de origen o destino según si es un flujo de mensajes Receiver o Sender West City Hospital
TCPIPNode El nombre de nodo y la dirección TCP/IP del nodo HL7 que utiliza el flujo de mensajes HL7 In/localhost:1111 o HL7 Out/localhost:2222
SourceIdentifier El identificador HL7 de la aplicación de origen Registro de paciente
SourceFacility El recurso HL7 de la aplicación de origen West City Hospital
DestinationIdentifier El identificador HL7 de la aplicación de destino Farmacia
DestinationFacility El recurso HL7 de la aplicación de destino Otro hospital urbano
QueuePrefix El prefijo de cola de este flujo de mensajes, o en blanco. Si el prefijo de cola está en blanco significa que se utiliza el nombre de instancia de patrón como prefijo de cola. ADTQ
