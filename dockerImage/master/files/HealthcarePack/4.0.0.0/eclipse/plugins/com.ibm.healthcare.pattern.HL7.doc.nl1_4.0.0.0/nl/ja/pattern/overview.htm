「Healthcare: HL7 to HL7」パターン

「Healthcare: HL7 to HL7」パターンは、Health Level Seven International (HL7) v2 メッセージを送信できるアプリケーションを、HL7 メッセージを受信できる 1 つ以上のアプリケーションと統合します。 それらのアプリケーションは、Minimal Lower Layer Protocol (MLLP) over TCP/IP を使用して HL7 メッセージを送信および受信できる必要があります。

注:「Healthcare: HL7 to HL7」パターンは IBM WebSphere Message Broker Connectivity Pack for Healthcare バージョン 7.0 で使用可能になり、このバージョンの IBM Integration Bus Healthcare Pack でも引き続き使用でき、サポートされています。 ただし、可能な場合は、代わりに「Healthcare: HL7 to HL7 DFDL」パターンを使用することをお勧めします。 「Healthcare: HL7 to HL7 DFDL」パターンは、MRM および HL7v25P メッセージ・セットの代わりに DFDL メッセージ・モデルを使用します。 DFDL メッセージ・モデルには、以下の利点があります。

  • DFDL がオープン・スタンダードなフォーマットであるのに対し、MRM および HL7v25P メッセージ・セットは IBM Integration Bus 独自のものです。
  • DFDL エディターは、MRM および HL7v25P メッセージ・セットと比べると、HL7 スキーマに対する拡張機能をより容易に開発およびテストできるツールを提供しています。
  • DFDL メッセージ・モデルが HL7 バージョン 2.7、2.6、および 2.5.1 以前をサポートしているのに対し、MRM および HL7v25P メッセージ・セットは HL7 バージョン 2.5.1 以前のみをサポートしています。

重要

パターン・インスタンスをデプロイする前に、HL7 メッセージ・セットを Integration ノードにデプロイする必要があります。 このメッセージ・セットは、HL7 メッセージがメッセージ・フローによって処理される際に、HL7 メッセージを解析するために使用されます。 パターンによって生成されるメッセージ・フローを Integration ノードにデプロイするには、その前にメッセージ・セットをデプロイしておく必要があります。

HL7 メッセージ・セットは、パターンのリソースのページからワークスペースにインポートできます。

HL7 メッセージ・セットは、ビルドするのに長時間かかり、ワークスペースの動作を著しく低速にします。 このため、以下のようにすることをお勧めします。

  • HL7 メッセージ・セットをワークスペースにインポートします。
  • HL7 メッセージ・セットを BAR ファイルに追加します。
  • BAR ファイルを Integration ノードにデプロイします。
  • HL7 メッセージ・セットと BAR ファイルをワークスペースから削除します。

メッセージ変換にグラフィカル・マッパーを使用する場合は、HL7 メッセージ・セットをワークスペース内に残しておく必要があります。 メッセージ・セットのビルドに要する時間を短縮するには、メッセージ・セットから HL7 チャプターを削除します。 ただし、メッセージ・セットに含まれる segments.mxsd、fields.mxsd および datatypes.mxsd の各メッセージ定義ファイルは、パターンに必要なため、残しておいてください。

オプション

「Healthcare: HL7 to HL7」パターンの受信側フローは、以下のオプションをサポートしています。

  • MLLP over TCP/IP を使用して HL7 メッセージを受信する。
  • メッセージを検証および解析する。
  • 着信メッセージを監査する。
  • 重複したメッセージをチェックし、処理する。
  • 順序付けオプションをサポートする。
  • 各着信メッセージをパターンの外部で処理するため、パブリッシュまたはキューへの送信を行う。
  • 変換および経路指定キューに出力を送信する。
  • 例外の処理

「Healthcare: HL7 to HL7」パターンの TransformAndRoute フローは、以下のオプションをサポートしています。

  • Canonical XML フォーマットに変換する。
  • パターンの外側で処理するために、着信する正規メッセージをパブリッシュするかキューに送信する。
  • HL7 セグメントに含まれる追加のフィールドについてレポートする。
  • メッセージを、独立して構成可能な複数の宛先に配布する。
  • メッセージをフィルタリングして、各宛先に送信する。
  • セグメントをフィルタリングして、各宛先に組み込む。
  • 非標準で宛先に固有の変更を組み込めるようにするカスタマイズ・ポイント。
  • HL7 に逆変換し、各宛先の送信側フロー用のキューに出力を送信する。
  • 例外の処理

「Healthcare: HL7 to HL7」パターンの各送信側フローは、以下のオプションをサポートしています。

  • この宛先用に変換されたメッセージを読み取る。
  • 変換されたメッセージを妥当性検査する。
  • 順序付けオプションに従ってメッセージが順番に送達されるようにする。
  • 順番どおりに到着しなかったメッセージがあれば通知する。
  • MLLP over TCP/IP を使用して HL7 メッセージ宛先アプリケーションに送達する。
  • 例外の処理
