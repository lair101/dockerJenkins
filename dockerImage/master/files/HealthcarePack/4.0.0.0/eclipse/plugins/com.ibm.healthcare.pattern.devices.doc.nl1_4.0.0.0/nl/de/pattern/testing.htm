Muster 'Healthcare: Medizinische Geräte an EKA' testen

Mit den folgenden Informationen können Sie das Muster 'Healthcare: Medizinische Geräte an EKA' testen.

Basiseinrichtung der Musterinstanz

Die wichtigsten Nachrichtenflüsse der Musterinstanz lauten wie folgt:

  • MedicalDevices
  • TransformAndRoute
  • Sender
  • PatientIdentifiers

Stellen Sie sicher, dass für die Musterinstanz alle Warteschlangen konfiguriert sind; Informationen finden Sie im Abschnitt "Warteschlangen" unter Musterinstanz 'Healthcare: Medizinische Geräte an EKA' verwalten.

Informationsfluss durch die Musterinstanz

Bei der Implementierung einer Instanz des Musters 'Healthcare: Medizinische Geräte an EKA' fließen die Informationen in der folgenden Reihenfolge:

  1. Der MedicalDeviceInput-Knoten sucht seinen konfigurierbaren Service, um festzustellen, zu welchen medizinischen Geräten eine Verbindung hergestellt werden muss, und welche Messungen erfasst werden müssen.
  2. Der MedicalDeviceInput-Knoten liest eine Messung des medizinischen Geräts und gibt diese Informationen in Form einer Nachricht an den Nachrichtenfluss 'MedicalDevices' weiter.
  3. Der Nachrichtenfluss 'MedicalDevices' fügt eine Ablaufzeit hinzu und sendet die Nachricht an die RXF-Warteschlange für den Nachrichtenfluss 'TransformAndRoute'.
  4. Der Nachrichtenfluss 'TransformAndRoute' konvertiert die Gerätenachricht in ein HL7-Überwachungsergebnis. Die Nachricht wird in einem kanonischen HL7-Format veröffentlicht, sodass sie von anderen Anwendungen verarbeitet werden kann. Das Überwachungsergebnis wird der aktuellen Patientenkennung für das betreffende Gerät zugeordnet und anschließend in die DEST-Warteschlange geschrieben.
  5. Der Nachrichtenfluss 'Sender' sendet die HL7-Nachricht an die EKA-Anwendung, die wiederum mit einer Bestätigung (ACK) antwortet.
  6. Die Bestätigungsnachricht wird vom Nachrichtenfluss 'Sender' verarbeitet.

Alle Flüsse verfügen auch über einen untergeordneten Nachrichtenfluss zur Ausnahmebedingungsbehandlung, der sämtliche Probleme bearbeitet, die vom Nachrichtenfluss abgefangen werden.

Die HL7-Testanwendung kann während der Entwicklung für die Simulation der empfangenden EKA-Anwendung verwendet werden. Starten Sie die Testanwendung und einen TCP/IP-MLLP-Listener. Die Testanwendung wird die von der Musterinstanz gesendeten Überwachungsergebnisse empfangen und anzeigen.

Führen Sie zum Starten der HL7-Testanwendung die folgenden Schritte aus:

  1. Klicken Sie auf Datei > Neu > Andere.
  2. Wählen Sie im Ordner Healthcare die Option HL7-Testanwendung aus und klicken Sie auf Weiter.
  3. Geben Sie einen Container und einen Dateinamen für die Testanwendungseinstellungen ein und klicken Sie dann auf Fertigstellen. Die HL7-Testanwendung wird gestartet.
