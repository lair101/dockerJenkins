Datenbank der Patienten-IDs einrichten

Der untergeordnete Nachrichtenfluss 'Healthcare: Medizinische Geräte an EKA' kann optional eine Datenbank auf Patienten-IDs abfragen. Die Kennzeichnung der Patienten ist wichtig, damit Gerätemessungen tatsächlich dem richtigen Patienten zugeordnet werden. Die Datenbank ordnet den Patientenkennungen bestimmte Gerätekennungen zu. Die Gerätekennungen werden in den Nachrichten vom MedicalDeviceInput-Knoten aus weitergegeben. Die Musterinstanz nimmt die Patientenkennung in die HL7-Überwachungsergebnisnachrichten auf, die an die Zielanwendung gesendet werden.

Zur Suche von Patientenkennungen benötigen Sie eine Datenbank mit einer Tabelle und einer ODBC-Definition. Damit ein Zugriff auf die Datenbank möglich ist, müssen für den Integrationsknoten eine Benutzer-ID und ein Kennwort konfiguriert sein.

Datenbanktabellen einrichten

Führen Sie zur Einrichtung der Datenbanktabellen folgende Tasks aus:

  1. Erstellen Sie die Datenbank.

    Befolgen Sie die für Ihren Datenbankmanager relevanten Anweisungen:

    • DB2 einrichten
    • Oracle einrichten

Datenbankumgebung für den Integrationsknoten einrichten

Führen Sie die folgenden Tasks aus, um die Datenbankumgebung für den Intergrationsknoten einzurichten:

  1. Erstellen Sie eine ODBC-Definition für die PATSDB-Datenbank mit PATIENTSDSN als Datenquellennamen (DSN); weitere Informationen hierzu finden Sie im Abschnitt 'ODBC-Verbindungen zu den Datenbanken herstellen' in der Dokumentation zu IBM Integration Bus.
  2. Geben Sie die Benutzer-ID und das Kennwort für den Datenquellennamen PATIENTSDSN .

    Geben Sie über den Befehl mqsisetdbparms eine Benutzer-ID und ein Kennwort an, die bzw. das der Integrationsknoten für die Datenquelle PATIENTSDSN verwenden kann. Diese Benutzer-ID muss mit der Benutzer-ID identisch sein, die Sie bei der Erstellung der Datenbank im vorherigen Schritt verwendet haben. Die Benutzer-ID legt den Datenbankschemanamen fest; der Integrationsknoten muss für den Zugriff auf die Tabelle dieselbe Benutzer-ID verwenden. Wenn Ihre Benutzer-ID beispielsweise USERID lautet, wird als vollständig qualifizierter Name der PATSDB-Tabelle der Tabellenname USERID.PATSDB verwendet.

    1. Öffnen Sie ein Befehlskonsolenfenster, in dem Sie IBM Integration Bus-Befehle ausführen können. Geben Sie zum Angeben der Benutzer-ID und des Kennworts den folgenden Befehl ein: 
      mqsisetdbparms Integrationsknoten -n PATIENTSDSN -u <Benutzer-ID> -p <Kennwort>
    2. Geben Sie den folgenden Befehl ein, um den Integrationsserver erneut zu laden: 
      mqsireload Integrationsknoten -e Name_des_Integrationsservers
      Dabei ist Integrationsknoten der Name des Integrationsknotens und Name_des_Integrationsservers der Name des Integrationsservers.
