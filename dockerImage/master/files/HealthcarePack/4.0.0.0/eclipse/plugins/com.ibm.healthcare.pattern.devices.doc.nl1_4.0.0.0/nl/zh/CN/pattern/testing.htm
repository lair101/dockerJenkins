测试“Healthcare：医疗设备到 EMR”模式

使用以下信息可帮助您测试“Healthcare：医疗设备到 EMR”模式。

基本模式实例设置

此模式实例的主要消息流如下所示：

  • MedicalDevices
  • TransformAndRoute
  • Sender
  • PatientIdentifiers

您必须确保已经为模式实例设置所有队列，请参阅管理“Healthcare：医疗设备到 EMR”模式实例中的“队列”部分。

模式实例中的信息流

部署“Healthcare：医疗设备到 EMR”模式的实例时，信息将按以下顺序流动：

  1. MedicalDeviceInput 节点找到其可配置服务，以确定要连接到哪些医疗设备以及要收集哪些测量。
  2. MedicalDeviceInput 读取医疗设备测量，并以消息形式将信息传播到 MedicalDevices 消息流中。
  3. MedicalDevices 消息流添加到期时间，并将消息发送到 TransformAndRoute 消息流的 RXF 队列。
  4. TransformAndRoute 消息流将设备消息转换为 HL7 观察结果。该消息以 HL7 规范格式发布，以供其他应用程序使用。观察结果与该设备的当前患者标识相关联，随后写入 DEST 队列。
  5. Sender 消息流将 HL7 消息发送到 EMR，后者以 ACK 进行响应。
  6. Sender 流对 ACK 消息进行处理。

所有的流都还有一个异常处理子流，用于处理该流所捕获到的所有问题。

在开发期间，可以使用 HL7 测试应用程序来模拟接收 EMR。请启动测试应用程序，并启动 TCP/IP MLLP 侦听器。此测试应用程序将接收并显示模式实例所发送的观察结果。

要启动 HL7 测试应用程序，请完成以下步骤：

  1. 单击文件 > 新建 > 其他。
  2. 在 Healthcare 文件夹中，选择 HL7 测试应用程序并单击下一步。
  3. 输入测试应用程序设置的容器和文件名，并单击完成。HL7 测试应用程序将开始。
