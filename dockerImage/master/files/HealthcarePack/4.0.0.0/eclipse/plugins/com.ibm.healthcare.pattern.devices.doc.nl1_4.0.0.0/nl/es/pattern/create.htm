Configuración de la base de datos de identificadores de pacientes

Opcionalmente, el patrón Servicio sanitario: Dispositivos médicos a EMR puede consultar los identificadores de pacientes en una base de datos. La identificación de pacientes es importante para que las mediciones de los dispositivos se asocien al paciente correcto. La base de datos asocia los identificadores de dispositivos con los identificadores de pacientes. Los identificadores de dispositivos se propagan en los mensajes desde el nodo MedicalDeviceInput. El identificador de paciente lo añade la instancia de patrón en los mensajes de resultados de observación HL7 v2 (ORU R01) que se envían a la aplicación de destino.

Necesita una base de datos con una tabla y una definición de ODBC para buscar identificadores de paciente. Para acceder a la base de datos, el nodo de integración debe estar configurado con un identificador de usuario y una contraseña.

Configurar las tablas de base de datos

Realice las tareas siguientes para configurar las tablas de base de datos:

  1. Cree la base de datos

    Siga las instrucciones relevantes para su gestor de base de datos:

    • Configuración de DB2
    • Configuración de Oracle

Configurar el entorno de base de datos para el nodo de integración

Realice las tareas siguientes para configurar el entorno de base de datos para el nodo de integración:

  1. Cree una definición de ODBC para la base de datos PATSDB con PATIENTSDSN como nombre de origen de datos (DSN); consulte Habilitación de las conexiones ODBC con las bases de datos en la documentación de IBM Integration Bus.
  2. Especifique el identificador de usuario y la contraseña asociados al nombre del origen de datos PATIENTSDSN .

    Utilice el mandato mqsisetdbparms para especificar un identificador de usuario y una contraseña que el nodo de integración pueda utilizar con el origen de datos PATIENTSDSN . Este identificador de usuario debe ser el mismo identificador de usuario que se ha utilizado al crear la base de datos en el paso anterior. El identificador de usuario determina el nombre del esquema de la base de datos, y el nodo de integración debe utilizar el mismo identificador de usuario al acceder a la tabla. Por ejemplo, si el identificador de usuario es USERID , el nombre completo de la tabla PATSDB es USERID.PATSDB .

    1. Abra una ventana de consola de mandatos donde pueda ejecutar mandatos de IBM Integration Bus. Escriba el siguiente mandato para especificar el identificador de usuario y la contraseña: 
      mqsisetdbparms nodo integración -n
PATIENTSDSN -u
<ID usuario> -p <contraseña>
    2. Escriba el siguiente mandato para volver a cargar el servidor de integración: 
      mqsireload nodo integración -e
nombre servidor integración
      donde nodo integración es el nombre del nodo de integración y nombre servidor integración es el nombre del servidor de integración.
