パターン・パラメーター デフォルト 説明
有効期限 (秒) 60

測定時に各医療デバイス・メッセージに設定される、秒単位で表した時間です。 デバイス・メッセージは、この時間が経過するまでに処理されなかった場合に、破棄される対象となります。

このパラメーターを空にすると、デバイス・メッセージはタイムアウトしなくなります。 古いデバイス・メッセージが、後続の処理時に間違った患者と偶発的に関連付けられることがないように、メッセージの有効期限を設定することをお勧めします。
ソース ID メッセージ・ブローカー このパラメーターは、エラー・メッセージ、通知、および標準のジャーナル・メッセージ内でのソースの記述に使用される ID を定義します。
ソース・ファシリティー このパラメーターは、エラー・メッセージ、通知、および標準のジャーナル・メッセージに含まれるソース・ファシリティーを定義します。
患者 ID デバイスからの患者 ID を使用します

このパラメーターは、患者 ID をデバイス・メッセージと関連付ける方法を定義します。 患者 ID を使用して、HL7 検査結果メッセージの PID (患者 ID) セグメントにデータを設定します。

  1. 検査結果に患者 ID がありません - HL7 検査結果メッセージに PID セグメントが不要であることを指定します。
  2. デバイスからの患者 ID を使用します - デバイスのために構成された患者 ID が、HL7 検査結果メッセージの PID セグメントに直接コピーされます。
  3. データベース内での患者 ID のルックアップ - デバイス・メッセージ内のデバイス ID をデータベース表へのキーとして使用することで、関連付けられた患者 ID を検索します。 そして患者 ID が、HL7 検査結果メッセージの PID セグメントにコピーされます。
メッセージ ID 現在の日付と時刻

このパラメーターは、HL7 検査結果 (ORU) メッセージ内で MSH.10 メッセージ ID フィールドが作成される方法を定義します。

「現在の日付と時刻」では、現在の日付と時刻を結合して、一連の数字にします。 この日付と時刻は、現行マシンで 0.000001 秒の精度です。 複数の Integration ノードが検査結果を同じ宛先アプリケーションに送信できるような高可用性の構成では、「現在の日付と時刻」は適していません。

「グローバル固有 ID」では、現在のタイム・スタンプと、現行マシンのネットワーク MAC アドレスなどの追加情報とに基づいて、英数字の ID を作成します。 「グローバル固有 ID」では、HL7 標準で推奨されている 20 文字制限より長い ID を作成します。
患者クラス 入院患者 I

患者クラスは、患者を場所別に分類するためにシステムで使用されます。 業界全体にわたる一貫性のある定義はありません。 現場固有の相違に左右されます。 選択されたパラメーター値は、HL7 ORU R01 Patient Visit Segment (PV1) の Patient Class フィールド (PV1-2) にデータを設定するために使用されます。

Web サービスの作成 選択

データ・ソース PATIENTSDSN

このパラメーターは、患者 ID が照会されるデータ・ソースの名前を定義します。

このパラメーターは読み取り専用です。 パターン用に構成する必要がある ODBC データ・ソース名を明らかにするために提供されます。
患者 ID テーブル PATIENTS

このパラメーターは、患者 ID とデバイス ID が含まれるテーブルを定義します。 パターン・インスタンスでは、デバイス・メッセージからのデバイス ID を使用して、このテーブルを照会します。 このテーブルでは、患者が関連付けられているデバイスごとに行が存在する必要があります。

このパラメーターは読み取り専用です。 パターン用に構成する必要があるデータベース表を明らかにするために提供されます。
宛先 EMR
パターン・パラメーター デフォルト 説明
宛先タイプ Generic このパラメーターは、宛先アプリケーションのタイプを定義します。 このパラメーターは、送信側フローで GenericHL7Output ノードを選択するために使用します。
宛先 ID EMR このパラメーターは、宛先の ID を定義します。 この ID は、宛先に関連したエラー、報告、および通知メッセージで使用されます。
宛先ファシリティー このパラメーターは、宛先のファシリティーを定義します。 このファシリティーは、宛先に関連したエラー、報告、および通知メッセージで使用されます。
接続の詳細 localhost:2222 このパラメーターは、宛先用の TCP/IP 接続を hostname:port 形式で定義します。
標準 MLLP 選択 このパラメーターは、TCP/IP および MLLP についてデフォルトの構成を使用するかどうかを決定します。 デフォルト構成の値は以下のとおりです。
  • タイムアウト: 60 秒
  • 先頭の MLLP バイト: 0B
  • 区切り文字: 適用外
  • 再試行限界: 3
  • 再試行をログに記録: true
  • 検証: false

このパラメーターの選択を解除すると、標準以外の値を定義するためのパラメーター・グループが表示されます。

宛先通信
パターン・パラメーター デフォルト 説明
タイムアウト (秒) 60 このパラメーターは、宛先のタイムアウトを秒単位で定義します。
先頭の MLLP バイト 0B このパラメーターは、出力レコードに追加される、先頭の MLLP バイトを定義します。
末尾の MLLP バイト 1C0D このパラメーターは、区切り文字が何であるかを定義します。 この値は 16 進数で指定する必要があります。
再試行限界 3 このパラメーターは、送信側フローが宛先アプリケーションへの HL7 メッセージの送信を試行する回数を定義します。
再試行をログに記録 選択 このパラメーターは、メッセージの送達試行をそれぞれログに記録するかどうかを決定します。
検証 なし このパラメーターは、HL7 メッセージ・セットに照らしてメッセージを検証するかどうかを指定します。
オプション
パターン・パラメーター デフォルト 説明
ジャーナリング 標準 このパラメーターは、ジャーナリングのオプションを決定します。有効な値は以下のとおりです。
  • なし: ジャーナリングはありません。 このオプションは、パターンのユーザーが、自身の要件に対するジャーナリングの実行のためにソース・フィードを使用する場合に使用することもできます。
  • 標準: 各メッセージのコピーが、パターンで定義された標準ヘッダー情報とともに生成されます。 このコピーは、「パブリッシュ」パラメーターの指定に従って、トピック PatternInstance/Journal にパブリッシュされるか、またはジャーナル・キューに書き込まれます。
正規のフィード クリア このパラメーターは、正規の形式でメッセージのコピーが生成されるかどうかを指定します。 選択された場合、「パブリッシュ」パラメーターによって、このメッセージのコピーが PatternInstance.CAN キューに送信されるか、それともトピック PatternInstance/Canonical に送信されるかが決まります。
パブリッシュ パブリッシュ このパラメーターは、ジャーナル・メッセージ、剰余メッセージ、およびその他のレポートが、パブリッシュされるか、キューに書き込まれるかを決定します。

説明 医療デバイスから EMR へのパターン このパラメーターは、生成されるフローの目的と必要に応じた任意の詳細を定義します。
キュー接頭部 このパラメーターは、キュー名の接頭部のストリングを定義します。標準キューは、この接頭部と一連の標準の接尾部付きで作成されます。

このパラメーターをブランクのままにすると、パターン・インスタンス名から非英数字を削除する処理が行われ、その処理されたパターン・インスタンス名がキューの接頭部として使用されます。
フロー接頭部 このパラメーターは、生成されるパターン・インスタンスの各メイン・メッセージ・フロー名の先頭に追加するストリングを定義します。このストリングは、レポート内およびモニターにおいて、パターン・インスタンスのコンポーネントを識別するのに役立ちます。

このパラメーターをブランクのままにすると、パターン・インスタンス名から非英数字を削除する処理が行われ、その処理されたパターン・インスタンス名がフローの接頭部として使用されます。
構成可能サービス接頭部 このパラメーターは、構成可能サービス名の接頭部のストリングを定義します。

このパラメーターをブランクのままにすると、パターン・インスタンス名から非英数字を削除する処理が行われ、その処理されたパターン・インスタンス名が構成可能サービス名として使用されます。
スクリプトの生成 選択 このパラメーターは、MQSC デプロイメント・スクリプトを生成するかどうかを決定します。
トレース・ファイル名 tracefile.txt このパラメーターは、トレース出力の送信先であるファイルの名前 (パスを含む) を定義します。パスを指定しない場合、トレース・ファイルは、IBM Integration Bus のインストール済み環境のマシン全体の作業パス・ディレクトリーに書き込まれます。Windows でのデフォルトの作業パス・ディレクトリーは C:¥ProgramData¥Application Data¥IBM¥MQSI です。Linux でのデフォルトの作業パス・ディレクトリーは /var/mqsi です。

すべてのパターン・パラメーターとそのデフォルト値をリストします。 デフォルトが示されていない場合、デフォルト値は空ストリングです。
