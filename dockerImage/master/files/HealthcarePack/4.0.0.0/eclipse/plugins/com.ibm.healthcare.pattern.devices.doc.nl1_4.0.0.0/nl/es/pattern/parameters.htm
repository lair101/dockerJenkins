Parámetros para el patrón Dispositivos médicos a EMR

Resultados de la observación
Parámetro de patrón Valor predeterminado Descripción
Tiempo de caducidad (segundos) 60

Es un período de tiempo expresado en segundos que se define en cada mensaje de dispositivo médico al medirlo. Los mensajes de dispositivo se podrán descartar si no se han procesado antes de que haya transcurrido este periodo de tiempo.

Si se deja este parámetro vacío, ningún mensaje de dispositivo excederá nunca el tiempo de espera. Se recomienda definir un tiempo de caducidad de los mensajes, de forma que los mensajes de dispositivo antiguos no se asocien nunca a un paciente incorrecto cuando se procesen más tarde.
Identificador de origen Intermediario de mensajes Este parámetro define un identificador que se utiliza para describir el origen en los mensajes de error, notificaciones y mensajes de registro estándar.
Recurso de origen Este parámetro define el recurso de origen que se incluye en los mensajes de error, notificaciones y mensajes de registro estándar.
Identificadores de pacientes Utilizar identificadores de pacientes del dispositivo

Este parámetro define cómo se asocian los identificadores de pacientes con los mensajes de dispositivo. El identificador de paciente se utiliza para cumplimentar el segmento PID (patient identifier) de los mensajes de resultados de observación de HL7:

  1. Ningún identificador de paciente en los resultados de observación: especifica que no se requiere ningún segmento PID en los mensajes de resultados de observación de HL7.
  2. Utilizar identificador de paciente del mensaje del dispositivo: el identificador de paciente configurado para el dispositivo se copia directamente en el segmento PID del mensaje de resultados de observación de HL7.
  3. Buscar el identificador de paciente en una base de datos: el identificador de paciente del mensaje del dispositivo se utiliza como clave en una tabla de base de datos para buscar el identificador de paciente asociado. A continuación, el identificador de paciente se copia al segmento PID del mensaje de resultados de observación de HL7.
Identificadores de mensajes Fecha y hora actuales

Este parámetro define cómo se crea el campo de identificador de mensaje MSH.10 en los mensajes de resultados de observación (ORU) de HL7.

Fecha y hora actuales combina la fecha y hora actuales en una secuencia de caracteres numéricos. La fecha y hora tiene una precisión de 0.000001 segundos respecto a la de la máquina actual. Fecha y hora actuales no es adecuado para configuraciones de alta disponibilidad donde más de un nodo de integración puede enviar resultados de observación a la misma aplicación de destino.

Identificador exclusivo global crea un identificador numérico alfa basado en la indicación de fecha y hora actual y otra información adicional como la dirección MAC de red de la máquina actual. Identificador exclusivo global crea identificadores más largos que el límite de 20 caracteres recomendado por el estándar HL7.
Clase de paciente Paciente entrante I

Los sistemas utilizan la clase de paciente para categorizar los pacientes por sitio. No tiene una definición unificada a nivel de todo el sector. Está sujeta a variaciones específicas según el sitio. El valor de parámetro seleccionado se utiliza para cumplimentar el campo Clase de paciente (PV1-2) en el Segmento de visita de paciente (PV1) ORU R01 de HL7.

Crear servicio web Seleccionado

Si este parámetro está seleccionado, se generan flujos adicionales para implementar un servicio web que gestiona la base de datos de identificadores de paciente.
Origen de datos PATIENTSDSN

Este parámetro define el nombre del origen de datos a partir de los cuales se consultan los identificadores de pacientes.

Este parámetro es de sólo lectura. Se presenta para resaltar el nombre del origen de datos ODBC que debe estar configurado para el patrón.
Tabla de identificadores de pacientes PATIENTS

Este parámetro define la tabla que contiene los identificadores de pacientes y de dispositivos. La instancia de patrón consulta esta tabla utilizando los identificadores de dispositivo de los mensajes de dispositivo. En la tabla, debe haber una fila para cada uno de los dispositivos que tenga asociado un paciente.

Este parámetro es de sólo lectura. Se presenta para resaltar la tabla de base de datos que debe estar configurada para el patrón.
EMR de destino
Parámetro de patrón Valor predeterminado Descripción
Tipo de destino Genérico Este parámetro define el tipo de aplicación de destino. El parámetro se utiliza para seleccionar el nodo GenericHL7Output en el flujo Sender.
Identificador de destino EMR Este parámetro define la identidad de un destino. Esta identidad se utiliza en los mensajes de error, de informe y de notificación relacionados con el destino
Recurso de destino Este parámetro define el recurso de un destino. Este recurso se utiliza en los mensajes de error, de informe y de notificación relacionados con el destino
Información detallada de conexión localhost:2222 Este parámetro define la conexión TCP/IP para un destino con el formato nombre_host:puerto
MLLP estándar Seleccionado Este parámetro determina si se utiliza una configuración predeterminada para TCP/IP y MLLP. La configuración predeterminada tiene los siguientes valores:
  • Tiempo de espera: 60 segundos
  • Byte MLLP inicial: 0B
  • Delimitador: No aplicable
  • Límite de reintentos: 3
  • Registrar de reintentos: true
  • Validación: false

Si el parámetro no está seleccionado, se hace visible el grupo de parámetros para definir valores no estándar.
Comunicaciones de destino
Parámetro de patrón Valor predeterminado Descripción
Tiempo de espera (segundos) 60 Este parámetro define el tiempo de espera para un destino en segundos.
Byte MLLP inicial 0B Este parámetro define el byte MLLP inicial, que se añade a los registros de salida.
Bytes MLLP finales 1C0D Este parámetro define el delimitador. Este valor debe ser hexadecimal.
Límite de reintentos 3 Este parámetro define el número de veces que un flujo Sender intenta entregar un mensaje HL7 a la aplicación de destino.
Registrar reintento Seleccionado Este parámetro determina si se registra cada uno de los intentos de entregar un mensaje.
Validación Ninguna Este parámetro determina si el mensaje se valida frente al conjunto de mensajes HL7.
Opciones
Parámetro de patrón Valor predeterminado Descripción
Registrar por diario Estándar Este parámetro determina las opciones de registro por diario. Los valores válidos son:
  • Ninguno: No registrar por diario. Esta opción también se puede utilizar si el usuario del patrón quiere utilizar un canal de información de origen para llevar a cabo el registro por diario para sus propios requisitos.
  • Estándar: Se genera una copia de cada mensaje con la información de cabecera estándar definida por el patrón. Esta copia se publica en el tema PatternInstance/Journal o se escribe en la cola de diario según determine el parámetro Publish.
Canal de información canónico Borrado Este parámetro determina si se genera una copia del mensaje en formato canónico. Si se selecciona, el parámetro Publish determina si esta copia del mensaje se envía a la cola o PatternInstance.CAN o al tema PatternInstance/Canonical
Publicar Publicar Este parámetro determina si los mensajes de diario, los mensajes restantes y otros informes se publican o se graban en colas.
General
Parámetro de patrón Valor predeterminado Descripción
Descripción Patrón Dispositivos médicos a EMR Este parámetro define la finalidad del flujo que se genera y puede incluir cualquier información detallada que desee.
Prefijo de cola Este parámetro define la serie de caracteres que desea prefijar a los nombres de cola. Las colas estándar se crean con este prefijo y un conjunto estándar de sufijos.

Si este parámetro se deja en blanco, se procesa el nombre de instancia de patrón eliminando los caracteres no alfanuméricos y se utiliza como prefijo de cola.
Prefijo de flujo Este parámetro define una serie de caracteres que se añade al principio del nombre de cada flujo de mensajes principal en una instancia de patrón generada. Esta serie le ayuda a identificar los componentes de la instancia de patrón en los informes y en la supervisión.

Si este parámetro se deja en blanco, se procesa el nombre de instancia de patrón eliminando los caracteres no alfanuméricos y se utiliza como prefijo del flujo.
Prefijo de servicio configurable Este parámetro define la serie de caracteres que desea prefijar a los nombres de servicio configurable.

Si este parámetro se deja en blanco, se procesa el nombre de instancia de patrón eliminando los caracteres no alfanuméricos y se utiliza como nombre del servicio configurable.
Generar scripts Seleccionado Este parámetro determina si se genera un script de despliegue MQSC.
Nombre de archivo de rastreo tracefile.txt Este parámetro define el nombre del archivo, incluyendo la vía de acceso, al que se envía la salida de rastreo. Si no se indica ninguna vía de acceso, el archivo de rastreo se graba en el directorio de vía de acceso de trabajo para toda la máquina de la instalación de IBM Integration Bus. En Windows, el directorio de vía de acceso de trabajo predeterminado es C:\ProgramData\Application Data\IBM\MQSI. En Linux, el directorio de vía de acceso de trabajo predeterminado es /var/mqsi.

Todos los parámetros de patrón aparecen en la lista con sus valores predeterminados. Cuando no se indica un valor predeterminado, éste es una serie vacía.
