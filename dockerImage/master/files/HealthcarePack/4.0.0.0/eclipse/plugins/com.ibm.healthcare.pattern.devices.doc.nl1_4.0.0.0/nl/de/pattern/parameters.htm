Parameter für das Muster Medizinische Geräte an EKA

Untersuchungsergebnisse
Musterparameter Standardeinstellung Beschreibung
Ablaufzeit (Sekunden) 60

Dies ist ein in Sekunden ausgedrückter Zeitraum, der bei ihrer Messung für jede Nachricht eines medizinischen Geräts festgelegt wird. Gerätenachrichten können gelöscht werden, wenn sie vor Ablauf dieses Zeitraums nicht verarbeitet wurden.

Wenn Sie diesen Parameter leer lassen, ist sichergestellt, dass eine Gerätenachricht das zulässige Zeitlimit niemals überschreitet. Es wird empfohlen, eine Ablaufzeit für Nachrichten festzulegen, damit veraltete Gerätenachrichten bei einer späteren Verarbeitung nicht versehentlich einem falschen Patienten zugeordnet werden.
Quellen-ID Message Broker Dieser Parameter definiert eine ID, mit der die Quelle in Fehlernachrichten, Benachrichtigungen und standardmäßigen Journaling-Nachrichten beschrieben wird.
Quelleneinrichtung Dieser Parameter definiert die Quellenfunktion, die in Fehlernachrichten, Benachrichtigungen und standardmäßigen Journaling-Nachrichten integriert ist.
Patientien-IDs Patienten-IDs aus Gerät verwenden

Dieser Parameter definiert, wie Patienten-IDs den Gerätenachrichten zugeordnet werden. Die Patienten-ID wird für die Belegung des PID-Segments (Patienten-ID) der HL7-Untersuchungsergebnisnachrichten verwendet:

  1. Keine Patienten-IDs in Untersuchungsergebnissen - Gibt an, dass in den HL7-Untersuchungsergebnisnachrichten kein PID-Segment erforderlich ist.
  2. Patienten-IDs aus Gerät verwenden - Die für das Gerät konfigurierte Patienten-ID wird direkt in das PID-Segment der HL7-Untersuchungsergebnisnachricht kopiert.
  3. In Datenbank nach Patienten-IDs suchen - Die Gerätekennung in der Gerätenachricht wird als Schlüssel für eine Datenbanktabelle verwendet, um die zugeordnete Patienten-ID zu suchen. Die Patienten-ID wird dann in das PID-Segment der HL7-Untersuchungsergebnisnachricht kopiert.
Nachrichten-IDs Aktuelles Datum und aktuelle Uhrzeit

Dieser Parameter definiert, wie das Feld "MSH.10" für die Nachrichtenkennung in den HL7-Untersuchungsergebnisnachrichten (ORU) erstellt wird.

Aktuelles Datum und aktuelle Uhrzeit kombiniert das aktuelle Datum und die aktuelle Uhrzeit zu einer Folge numerischer Zeichen. Das Datum und die Uhrzeit sind auf der aktuellen Maschine bis auf0,000001 Sekunden genau angegeben. Aktuelles Datum und aktuelle Uhrzeit ist für Konfigurationen mit hoher Verfügbarkeit nicht geeignet, in denen mehrere Integrationsknoten Untersuchungsergebnisse an ein und dieselbe Zielanwendung senden können.

Globale eindeutige ID (GUID) erstellt eine alphanumerische Kennung auf Basis der aktuellen Zeitmarke und auf der Grundlage zusätzlicher Informationen, zu denen auch die MAC-Netzadresse der aktuellen Maschine gehört. Mit der Einstellung Globale eindeutige ID (GUID) werden Kennungen erstellt, die den vom HL7-Standard empfohlenen Grenzwert von 20 Zeichen überschreiten.
Patientenklasse Stationärer Patient S

Die Patientenklasse wird von Systemen zur Kategorisierung von Patienten nach Standort verwendet. Sie ist in der Branche nicht einheitlich definiert, sondern variiert je nach Standort. Mit dem ausgewählten Parameterwert wird das Feld für die Patientenklasse (PV1-2) im Segment für die Patientenvisite (PV1) der HL7-ORU-R01-Nachricht belegt.

Web-Service erstellen Ausgewählt

Wenn dieser Parameter ausgewählt ist, werden zusätzliche Flüsse generiert, um einen Web-Service zu implementieren, der die Datenbank mit den Patienten-IDs verwaltet.
Datenquelle PATIENTSDSN

Dieser Parameter definiert den Datenquellennamen, über den Patienten-IDs abgefragt werden.

Dieser Parameter ist schreibgeschützt und wird dargestellt, um den ODBC-Datenquellennamen hervorzuheben, der für das Muster konfiguriert werden muss.
Tabelle mit Patienten-IDs PATIENTS

Dieser Parameter definiert die Tabelle, die die Patienten- und Gerätekennungen enthält. Die Musterinstanz führt unter Verwendung der Gerätekennungen aus den Gerätenachrichten eine Abfrage in dieser Tabelle durch. Diese Tabelle muss für jedes Gerät, dem ein Patient zugeordnet ist, eine Zeile enthalten.

Dieser Parameter ist schreibgeschützt und wird dargestellt, um die Datenbanktabelle hervorzuheben, die für das Muster konfiguriert werden muss.
EKA-Zielanwendung
Musterparameter Standardeinstellung Beschreibung
Zieltyp Generisch Dieser Parameter definiert den Typ der Zielanwendung. Der Parameter wird für die Auswahl des GenericHL7Output-Knotens im Nachrichtenfluss 'Sender' verwendet.
Ziel-ID EKA Dieser Parameter definiert die Identität eines Ziels. Diese Identität wird in Fehlernachrichten, Berichtsnachrichten und Benachrichtigungen verwendet, die mit dem Ziel in Zusammenhang stehen.
Zieleinrichtung Dieser Parameter definiert die Einrichtung eines Ziels. Diese Einrichtung wird in Fehlernachrichten, Berichtsnachrichten und Benachrichtigungen verwendet, die mit dem Ziel in Zusammenhang stehen.
Verbindungsdetails localhost:2222 Dieser Parameter definiert die TCP/IP-Verbindung für ein Ziel und wird in der Form Hostname:Port angegeben.
Standard-MLLP Ausgewählt Dieser Parameter legt fest, ob eine Standardkonfiguration für TCP/IP und MLLP verwendet wird. Die Standardkonfiguration weist folgende Werte auf:
  • Zeitlimit: 60 Sekunden
  • Führendes MLLP-Byte: 0B
  • Begrenzer: Nicht zutreffend
  • Wiederholungslimit: 3
  • Wiederholung protokollieren: true
  • Validierung: false

Wenn der Parameter nicht ausgewählt wird, wird die Parametergruppe für die Definition der vom Standard abweichenden Werte angezeigt.
Zielübertragungen
Musterparameter Standardeinstellung Beschreibung
Zeitlimit (Sekunden) 60 Dieser Parameter definiert in Sekunden das Zeitlimit für ein Ziel.
Führendes MLLP-Byte 0B Dieser Parameter definiert das führende MLLP-Byte, das den ausgehenden Datensätzen hinzugefügt wird.
Abschließende MLLP-Bytes 1C0D Dieser Parameter definiert den Begrenzer. Dieser Wert muss im hexadezimalen Format angegeben werden.
Wiederholungslimit 3 Dieser Parameter definiert, wie oft der Nachrichtenfluss 'Sender' versucht, eine HL7-Nachricht an die Zielanwendung zuzustellen.
Wiederholung protokollieren Ausgewählt Dieser Parameter legt fest, ob jeder Versuch einer Nachrichtenzustellung protokolliert wird.
Validierung Keine Dieser Parameter legt fest, ob die Nachricht anhand der HL7-Nachrichtengruppe validiert wird.
Optionen
Musterparameter Standardeinstellung Beschreibung
Journaling Standard Dieser Parameter legt die Optionen zur Journalführung fest. Folgendes sind gültige Werte:
  • Keine: Keine Journalführung. Diese Option kann auch verwendet werden, wenn der Benutzer eines Musters einen Quellen-Feed verwenden möchte, um die Journalführung für seine eigenen Anforderungen auszuführen.
  • Standard: Es wird eine Kopie jeder Nachricht mit den vom Muster definierten standardmäßigen Kopfzeileninformationen erstellt. Diese Kopie wird im Thema PatternInstance/Journal veröffentlicht oder in die Journalwarteschlange geschrieben (je nach Festlegung des Parameters Publish).
Kanonischer Feed Nicht ausgewählt Dieser Parameter legt fest, ob eine Nachrichtenkopie im kanonischen Format generiert wird. Ist der Parameter Publish ausgewählt, legt er fest, ob diese Nachrichtenkopie an die Warteschlange PatternInstance.CAN oder an das Thema PatternInstance/Canonical gesendet wird.
Veröffentlichen Veröffentlichen Dieser Parameter legt fest, ob Journalnachrichten, Restnachrichten (Remainder) und sonstige Berichte veröffentlicht oder in Warteschlangen geschrieben werden.
Allgemein
Musterparameter Standardeinstellung Beschreibung
Beschreibung Muster 'Medizinische Geräte an EKA' Dieser Parameter definiert den Zweck des generierten Nachrichtenflusses und kann alle von Ihnen geforderten Einzelheiten enthalten.
Warteschlangenpräfix Dieser Parameter definiert eine Zeichenfolge, die Sie den Warteschlangennamen als Präfix voranstellen möchten. Die Standardwarteschlangen werden mit diesem Präfix und einer Standardgruppe von Suffixen erstellt.

Wenn dieser Parameter leer ist, wird der Name der Musterinstanz für das Entfernen von nicht alphanumerischen Zeichen verarbeitet und anschließend als Warteschlangenpräfix verwendet.
Nachrichtenflusspräfix Dieser Parameter definiert eine Zeichenfolge, die an den Anfang des Namens jedes Hauptnachrichtenflusses in einer generierten Musterinstanz hinzugefügt wird. Diese Zeichenfolge erleichtert die Ermittlung der Musterinstanz in Berichten und in der Überwachung.

Wenn dieser Parameter leer ist, wird der Name der Musterinstanz für das Entfernen von nicht alphanumerischen Zeichen verarbeitet und anschließend als Nachrichtenflusspräfix verwendet.
Präfix des konfigurierbaren Service Dieser Parameter definiert eine Zeichenfolge, die Sie als Präfix für den Namen des konfigurierbaren Service verwenden möchten.

Erfolgt für diesen Parameter keine Angabe, wird der Musterinstanzname so verarbeitet, dass nicht alphanumerische Zeichen entfernt werden. Anschließend wird er als Name des konfigurierbaren Service verwendet.
Scripts generieren Ausgewählt Dieser Parameter gibt an, ob ein MQSC-Implementierungsscript generiert wird.
Name der Tracedatei tracefile.txt Dieser Parameter definiert den Dateinamen einschließlich des Pfads, an den die Traceausgabe gesendet wird. Ist kein Pfad angegeben, wird die Tracedatei in das systemweite Arbeitspfadverzeichnis Ihrer IBM Integration Bus-Installation geschrieben. Unter Windows lautet das standardmäßige Arbeitspfadverzeichnis C:\ProgramData\Application Data\IBM\MQSI. Unter Linux ist das standardmäßige Arbeitspfadverzeichnis /var/mqsi.

Für alle Musterparameter sind die Standardwerte aufgeführt. Ist kein Standardwert angegeben, ist der Standardwert eine leere Zeichenfolge.
