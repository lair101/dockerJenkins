“医疗设备到 EMR”模式的参数

观察结果
模式参数 缺省值 描述
到期时间（秒） 60

这是对每条医疗设备消息进行测量时，对这些消息设置的时间段（以秒计）。如果设备消息在这段时间经过前未进行处理，那么将符合废弃条件。

将此参数保留为空可以确保设备消息永远不超时。建议您设置消息到期时间，以免到期的设备消息在后续处理时意外地与不正确的患者相关联。
源标识 消息代理 此参数定义一个标识，用于在错误消息、通知和标准日志记录消息中描述源。
源设施 此参数定义要包括在错误消息、通知和标准日志记录消息中的源设施。
患者标识 使用来自设备的患者标识

此参数定义患者标识与设备消息相关联的方式。患者标识用于填充 HL7 观察结果消息的 PID（患者标识）段：

  1. 观察结果中没有患者标识 - 指定 HL7 观察结果消息中不需要任何 PID 段。
  2. 使用来自设备消息的患者标识 - 为设备配置的患者标识将直接复制到 HL7 观察结果消息的 PID 段中。
  3. 在数据库中查找患者标识 - 设备消息中的设备标识用作数据库表的键，以查找相关联的患者标识。然后，患者标识将复制到 HL7 观察结果消息的 PID 段中。
消息标识 当前日期和时间

此参数定义如何在 HL7 观察结果 (ORU) 消息中创建 MSH.10 消息标识字段。

“当前日期和时间”将当前日期和时间组合为数字字符序列。在当前机器上，日期和时间精确到 0.000001 秒。对于多个集成节点可以向同一个目标应用程序发送观察结果的高可用性配置，“当前日期和时间”不适用。

“全局唯一标识”根据当前时间戳记和其他信息（包括当前机器的网络 MAC 地址）创建字母数字标识。“全局唯一标识”将创建超出 HL7 标准所建议的 20 字符长度限制的标识。
患者类别 住院患者 I

系统使用患者类按地点对患者进行分类。患者类没有一致的行业范围定义，而是可以随地点不同而有所变化。所选参数值用于填充 HL7 ORU R01 患者就诊段 (PV1) 中的“患者类”字段 (PV1-2)。
查找患者标识
模式参数 缺省值 描述
创建 Web 服务 选中

如果选中此参数，那么将生成其他的流，以实现用于管理患者标识数据库的 Web 服务。
数据源 PATIENTSDSN

此参数定义数据源名称，系统将在该数据源中查询患者标识。

此参数为只读 - 显示此参数是为了突出显示必须为此模式配置的 ODBC 数据源名称。
患者标识表 PATIENTS

此参数定义包含患者标识和设备标识的表。模式实例使用设备消息中的设备标识来查询这个表。对于每个与患者相关联的设备，在这个表中都应该存在相应的一行。

此参数为只读 - 显示此参数是为了突出显示必须为此模式配置的数据库表。
目标 EMR
模式参数 缺省值 描述
目标类型 通用 此参数定义目标应用程序的类型。此参数用于选择“发送方”流中的 GenericHL7Output 节点。
目标标识 EMR 此参数定义目标的身份。在与目标相关的错误、报告和通知消息中，将使用此身份。
目标设施 此参数定义目标的设施。在与目标相关的错误、报告和通知消息中，将使用此设施。
连接详细信息 localhost:2222 此参数以 hostname:port 格式定义目标的 TCP/IP 连接。
标准 MLLP 选中 此参数确定是否使用 TCP/IP 和 MLLP 的缺省配置。缺省配置使用下列值：
  • 超时：60 秒
  • 前导 MLLP 字节：0B
  • 定界符：不适用
  • 重试限制：3
  • 记录重试：true
  • 验证：false

如果未选中此参数，那么用于定义非标准值的参数组将变为可见。
目标通信
模式参数 缺省值 描述
超时（秒） 60 此参数定义目标的超时值（以秒计）。
前导 MLLP 字节 0B 此参数定义对出局记录添加的前导 MLLP 字节。
结尾 MLLP 字节 1C0D 此参数定义定界符。此值必须采用十六进制格式。
重试限制 3 此参数定义“发送方”流尝试将 HL7 消息交付给目标应用程序的次数。
记录重试 选中 此参数确定是否对每次交付消息的尝试进行记录。
验证 无 此参数确定是否根据 HL7 消息集对消息进行验证。
选项
模式参数 缺省值 描述
日志记录 标准 此参数确定日志记录选项。有效值包括：
  • 无：不进行日志记录。如果模式用户希望使用源订阅源对自己的需求执行日志记录，那么也可以使用此选项。
  • 标准：使用此模式定义的标准头信息生成每条消息的副本。此副本将发布到 PatternInstance/Journal 主题，或者写入由“发布”参数确定的日志队列。
规范订阅源 取消选中 此参数确定是否生成采用规范格式的消息副本。如果选中此参数，那么“发布”参数确定此消息副本是发送到 PatternInstance.CAN 队列还是发送到 PatternInstance/Canonical 主题。
发布 发布 此参数确定是发布日志消息、其余消息和其他报告，还是将这些消息和报告写入队列。
常规
模式参数 缺省值 描述
描述 医疗设备到 EMR 模式 此参数定义所生成的流的用途，并且可以包括您需要的任何详细信息。
队列前缀 此参数定义要用作队列名称前缀的字符串。标准队列将使用此前缀和标准后缀集进行创建。

如果将此参数保留为空，那么将对模式实例名进行处理以除去非字母数字字符，然后将处理后的模式实例名用作队列前缀。
流前缀 此参数定义一个字符串，此字符串将添加到生成的模式实例中每个主消息流的名称开头。此字符串可以帮助在报告和监视环境中标识模式实例的组件。

如果将此参数保留为空，那么将对模式实例名进行处理以除去非字母数字字符，然后将处理后的模式实例名用作流前缀。
可配置的服务前缀 此参数定义要用作可配置服务名称前缀的字符串。

如果将此参数保留为空，那么将对模式实例名进行处理以除去非字母数字字符，然后将处理后的模式实例名用作可配置服务名称。
生成脚本 选中 此参数确定是否生成 MQSC 部署脚本。
跟踪文件名 tracefile.txt 此参数定义跟踪输出所要发送到的文件的名称（包括路径）。如果未提供路径，那么跟踪文件将写入 IBM Integration Bus 安装的整个机器的工作路径目录。在 Windows 上，缺省工作路径目录为 C:\ProgramData\Application Data\IBM\MQSI。在 Linux 上，缺省工作路径目录为 /var/mqsi。

所有模式参数均随其缺省值一起列出。 未指示缺省值时，表明缺省值是空字符串。
