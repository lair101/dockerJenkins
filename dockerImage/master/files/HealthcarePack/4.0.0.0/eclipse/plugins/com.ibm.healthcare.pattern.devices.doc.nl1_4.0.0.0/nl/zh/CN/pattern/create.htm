设置患者标识数据库

“Healthcare：医疗设备到 EMR”模式可以选择性地查询数据库以获取患者标识。患者标识用于使设备测量与正确的患者相关联，因此非常重要。此数据库用于使设备标识与患者标识相关联。设备标识在来自 MedicalDeviceInput 节点的消息中进行传播。患者标识由模式实例添加到已发送至目标应用程序的 HL7 V2 观察结果消息 (ORU R01) 中。

您需要一个其中包含一个表的数据库以及 ODBC 定义，以便查找患者标识。要访问此数据库，必须为集成节点配置用户标识和密码。

设置数据库表

要设置数据库表，请完成以下任务：

  1. 创建数据库

    按照与数据库管理器相关的指示信息执行操作：

    • 设置 DB2
    • 设置 Oracle

为集成节点设置数据库环境

要为集成节点设置数据库环境，请完成下列任务：

  1. 为 PATSDB 数据库创建使用 PATIENTSDSN 作为数据源名称 (DSN) 的 ODBC 定义；请参阅 IBM Integration Bus 文档中的“启用与数据库的 ODBC 连接”。
  2. 指定与数据源名称 PATIENTSDSN 相关联的用户标识和密码。

    使用 mqsisetdbparms 命令指定可供集成节点用于 PATIENTSDSN 数据源的用户标识和密码。此用户标识必须是您在上一步中创建数据库时使用的用户标识。此用户标识确定了数据库模式名，集成节点在访问表时，必须使用同一用户标识。 例如，如果您的用户标识为 USERID，那么 PATSDB 表的标准名称为 USERID.PATSDB。

    1. 打开命令控制台窗口，您可以从中运行 IBM Integration Bus 命令。请输入以下命令以指定用户标识和密码：
      mqsisetdbparms integration node -n PATIENTSDSN -u <user ID> -p <password>
    2. 输入以下命令以重新装入集成服务器：
      mqsireload integration node -e integration server name
      其中，integration node 是集成节点的名称，而 integration server name 是集成服务器的名称。
