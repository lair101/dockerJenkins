Tasks to complete after generating the Healthcare: Home Health pattern

After you generate an instance of the Healthcare: Home Health pattern, complete the following steps:

  • Implement the user-defined subflow in the Healthcare: Home Health pattern:
    • The Healthcare: Home Health pattern uses UserDefinedFunction.subflow to process your HL7 v2.6 messages. If you want to customize how the pattern processes these messages go to Subflows/UserDefinedFunction.subflow.

      After the processing of the incoming message is complete, an acknowledgment is returned to the calling application or device.

  • Optional: If you expect the Healthcare: Home Health pattern to receive a large volume of requests, configure your system to use an appropriate HTTP topology. See Configuring your HTTP topology in the IBM Integration Bus documentation.
  • Optional: If you configure the Healthcare: Home Health pattern to enable SAML 2.0 token pass-through, complete the following steps:
    1. Modify and run the example configurable service scripts provided in the SecurityProfile scripts folder to define a security profile for an external SAML Secure Token Service. For more information, see SAML token capabilities for authentication and authorization in the IBM Integration Bus documentation.
    2. Create a BAR file that contains the Healthcare: Home Health application.
    3. In the BAR file editor, configure the SecurityPEP node with the security profile you defined in step 1. For more information, see Configuring the extraction of an identity or security token in the IBM Integration Bus documentation.
