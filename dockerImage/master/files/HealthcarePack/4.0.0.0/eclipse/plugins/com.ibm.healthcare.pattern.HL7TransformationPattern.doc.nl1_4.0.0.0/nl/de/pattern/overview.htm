Muster 'Healthcare: HL7-Transformation'

Das Muster 'Healthcare: HL7-Transformation' generiert grafische Datenzuordnungen, mit denen Sie HL7-Nachrichten assemblieren können.

HL7-Nachrichten enthalten für gewöhnlich viele Datensegmente. Die Segmente selbst enthalten wiederum eine große Anzahl einzelner Felder. Wenn Sie eine neue HL7-Nachricht erstellen, müssen Sie die HL7-Nachrichtenhierarchie erstellen. Diese Hierarchie kann unter Umständen Hunderte oder Tausende von Feldern enthalten. Ihre Erstellung kann recht zeitaufwändig sein, selbst wenn Sie ein grafisches Mapping-Tool verwenden. Das Muster 'Healthcare: HL7-Transformation' hilft, dieses Problem zu lösen, indem grafische Datenzuordnungen generiert werden, die allen Feldern in den von Ihnen ausgewählten HL7-Nachrichten einen Standardwert zuweisen.

Lösung

Das Muster 'Healthcare: HL7-Transformation' kann wie folgt verwendet werden:

  • Wählen Sie bestimmte HL7-Nachrichten aus. Bei jeder Auswahl werden folgende Artefakte generiert:
    • Eine grafische Datenzuordnung
    • Eine untergeordnete Zuordnung für jede HL7-Segmentdefinition
    Alle generierten Artefakte werden in einer Bibliothek namens Musterinstanzname_HL7v27TransformLibrary gespeichert, wobei Musterinstanzname_ den Namen Ihrer Musterinstanz angibt. Sie können die untergeordneten HL7-Segmentzuordnungen aus mehreren übergeordneten Zuordnungen abrufen.
  • Wählen Sie bestimmte HL7-Segmente aus. Jede Auswahl generiert eine untergeordnete Zuordnung der grafischen Datenzuordnung, die in der BibliothekMusterinstanzname_HL7v27TransformLibrary gespeichert wird. Sie können diese untergeordneten Segmentzuordnungen aus Ihren eigenen grafischen Datenzuordnungen abrufen, die Sie außerhalb des Bereichs dieses Musters erstellen.

Die Musterparameter bieten folgende Optionen:

  • Sie können grafische Datenzuordnungen oder Transformationslogik für Compute-Knoten (ESQL) generieren.
  • Sie können den Wert auswählen, der einzelnen Feldern in der HL7-Nachricht zugeordnet werden soll.
  • Sie können einzelne Werte für die Felder im HL7- Nachrichtenheadersegment (Message Header, MSH) festlegen.
  • Sie können einen Beispielnachrichtenfluss erstellen, der die Verwendung der Transformationslogik veranschaulicht.

Wenn Sie eine Instanz des Musters 'Healthcare: HL7-Transformation' generieren, wird das HL7-Nachrichtenmodell automatisch in Ihren Arbeitsbereich in der Bibliothek HL7v27DFDLLibrary importiert. Falls diese Bibliothek bereits vorhanden ist, wird eine Warnung angezeigt. Falls Sie sich für die Fortsetzung des Vorgangs entscheiden, wird die bestehende Bibliothek überschrieben. Wenn Sie Ihr HL7-Nachrichtenmodell anpassen möchten, nehmen Sie Ihre Änderungen vor, nachdem Sie eine Instanz des Musters generiert haben.

Wenn Sie den Musterparameter für die Erstellung eines Beispielnachrichtenflusses auswählen, enthält er drei Knoten:

  • Einen MQInput-Knoten, der Eingabenachrichten aus einer Warteschlange namens Musterinstanzname.RXF1 entnimmt. Dabei steht Musterinstanzname für den Namen Ihrer Musterinstanz.
  • Entweder einen Mapping- oder Compute-Knoten, je nachdem, welche Auswahl Sie für den Musterparameter Generate (Generieren) getroffen haben.
  • Einen MQOutput-Knoten, der Ausgabenachrichten in eine Warteschlange namens Musterinstanzname.DEST1 schreibt. Dabei steht Musterinstanzname für den Namen Ihrer Musterinstanz.

Ein Script wird generiert, das Sie zur Erstellung der vom Nachrichtenfluss verwendeten Warteschlangen ausführen können.

Wenn das Muster 'Healthcare: HL7-Transformation' für die Generierung der Transformationslogik zur Erstellung von HL7-Nachrichten verwendet wird, die eine Auswahlstruktur enthalten, stellen die generierten grafischen Datenzuordnungen oder Extended SQL (ESQL) eine untergeordnete Zuordnung bzw. einen Prozeduraufruf zur Verfügung. Die untergeordnete Zuordnung bzw. der Prozeduraufruf erstellt das erste untergeordnete Element der Auswahlstruktur. Bei den grafischen Datenzuordnungen werden mithilfe von Task-Transformationen Platzhalter für andere Auswahloptionen bereitgestellt. Bei Extended SQL (ESQL) werden mithilfe von kommentierten Prozeduraufrufen Platzhalter für andere Auswahloptionen bereitgestellt.
