Patrón Servicio sanitario: Transformación de HL7

El patrón Servicio sanitario: Transformación de HL7 genera correlaciones de datos gráficos que se pueden utilizar para ensamblar mensajes de HL7.

Los mensajes HL7 generalmente contienen muchos segmentos de datos. Los propios segmentos también pueden contener muchos campos individuales. Al crear un nuevo mensaje HL7, debe crear la jerarquía de mensajes HL7. Esta jerarquía puede contener cientos o miles de campos. Crear esta jerarquía puede llevar algún tiempo, aunque utilice una herramienta de correlación gráfica. El patrón Servicio sanitario: Transformación de HL7 le ayuda a solventar este problema generando correlaciones de datos gráficos que asignan un valor predeterminado a todos los campos de los mensajes HL7 que selecciona.

Solución

Puede utilizar el patrón Servicio sanitario: Transformación de HL7 de los siguientes modos:

  • Seleccionar mensajes HL7 específicos. Cada selección genera los siguientes artefactos:
    • Una correlación de datos gráficos
    • Una subcorrelación para cada definición de segmento HL7
    Todos los artefactos generados se ponen en una biblioteca denominada nombre_instancia_patrón_HL7v27TransformLibrary, donde nombre_instancia_patrón_ es el nombre de la instancia de patrón. Puede llamar a las subcorrelaciones de segmentos HL7 desde las diversas correlaciones de nivel superior.
  • Seleccionar segmentos HL7 específicos. Cada selección genera una subcorrelación de tipo Correlación de datos gráficos que se guarda en la biblioteca nombre_instancia_patrón_HL7v27TransformLibrary. Puede llamar a estas subcorrelaciones de segmentos desde sus propias Correlaciones de datos gráficos que haya creado fuera de este patrón.

Los parámetros de patrón proporcionan las opciones siguientes:

  • Puede generar Correlaciones de datos gráficos o lógica de transformación de nodos de cálculo (ESQL).
  • Puede elegir el valor a asignar a los campos individuales del mensaje HL7.
  • Puede establecer valores individuales para los campos del segmento MSH (HL7 Message Header).
  • Puede crear un flujo de mensajes de ejemplo que muestre cómo utilizar la lógica de transformación.

Cuando se genera una instancia del patrón Servicio sanitario: Transformación de HL7, se importa automáticamente el modelo de mensaje HL7 del espacio de trabajo a la biblioteca HL7v27DFDLLibrary. Se muestra un aviso si la biblioteca existe. Si elige continuar, se sobrescribe la biblioteca existente. Si desea personalizar el modelo de mensaje HL7, haga los cambios después de haber generado una instancia del patrón.

Si selecciona que el parámetro de patrón cree un flujo de mensajes de ejemplo, éste contiene tres nodos:

  • Un nodo MQInput que acepta mensajes de entrada de una cola denominada nombre_instancia_patrón.RXF1, donde nombre_instancia_patrón es el nombre de la instancia de patrón.
  • Un nodo Mapping o bien un nodo Compute, según lo que haya seleccionado en el parámetro de patrón Generar.
  • Un nodo MQOutput que graba los mensajes de salida a una cola denominada nombre_instancia_patrón.DEST1, donde nombre_instancia_patrón es el nombre de la instancia de patrón.

Se genera un script que puede ejecutar para crear las colas que utiliza el flujo de mensajes.

Si se utiliza el patrón Servicio sanitario: Transformación de HL7 para generar lógica de transformación para la creación de mensajes HL7 que contengan una estructura de opciones, las correlaciones de datos gráficos o el ESQL proporcionan una subcorrelación o una llamada de procedimiento respectivamente. La subcorrelación o la llamada de procedimiento crea el primer hijo de la estructura de opciones. En el caso de correlaciones de datos gráficos, se utilizan transformaciones de tareas para proporcionar marcadores para otras opciones. En el caso de ESQL, se utilizan llamadas de procedimiento comentadas para proporcionar marcadores para otras opciones.
