Constraints on the use of the Healthcare: FHIR Transformation pattern

The following considerations apply when using the Healthcare: FHIR Transformation pattern to convert HL7® FHIR® standard Resources from XML to JSON:

  • Comments in Resources in the XML format are ignored and are not transformed into the JSON format.
  • The transformation assumes an XML element without a 'value' attribute is complex and creates a JSON object.
  • Resources in the XML format are validated against the schema when they are read from disk, but the transformation is generic and does not use the schemas.

The following considerations apply when using the Healthcare: FHIR Transformation pattern to convert HL7® FHIR® standard Resources from JSON to XML:

  • The XML schemas are used to validate the Resource in XML format created by the transformation.
  • Properties in Resources in the JSON format do not have order. The transformation is generic and does not use the XML schemas, so JSON properties will appear in the transformed XML file in the same order as the original JSON file. If the order of the properties in the JSON file does not match the order in the schema, the validation will fail.
  • Comments in Resources in the JSON format are ignored and are not transformed into the XML format.
