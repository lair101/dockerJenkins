Parameters for the HL7 Transformation pattern

General Configuration Options
Pattern parameter Default Description
Derive Library from HL7 Messages

This parameter specifies whether you would like to generate the library contents based on HL7 Message definitions, or based on individual HL7 Segment definitions (which you can then combine to generate your own transformations).
Assignment Value

This parameter specifies a string value to be assigned to fields within the HL7 messages.
Generate Graphical Data Map

This parameter controls whether the generated library should contain Graphical Data Maps or ESQL code.
Construct MSH fields individually Selected

Check this option if you would like to generate an MSH segment transformation with different assigned values for each field. If this option is unchecked, then the MSH segment will be treated the same as other segments and the assigned values for each field will be the same.
Create an example Message Flow Selected

Check this option if you would like to generate an example message flow which uses the generated transformation logic. If you are deriving your library from HL7 Segments then this option will be ignored.
