Parameters for the Patient Demographics Query Supplier pattern

Connection Details
Pattern Parameter Default Description
Connection details localhost:1111 This parameter defines the TCP/IP connection details of the Patient Demographics Supplier. Configure your clinical application that is acting as the Patient Demographics Consumer in the PDQ profile to connect to this address. This parameter is in the form hostname:port
MDM Patient Hub (EMPI)
Pattern Parameter Default Description
Username This parameter specifies the username that is required to connect to the EMPI.
Password This parameter specifies the password that is required to connect to the EMPI.
Web Service URL http://localhost:9080/ibminitiatews/services/IdentityHub This parameter specifies the web service URL that is required to connect to the EMPI identity hub, for example: http://localhost:9080/ibminitiatews/services/IdentityHub
Member Type ASMEMBER This parameter determines whether linked members are returned in the query result. Select one of the following values:
  • ASMEMBER returns only members that match your search criteria
  • ASENTITY returns members that match the search criteria and any members that are linked to those matching members.
Minimum Score 0 This parameter defines the threshold value that is used to determine whether a member is returned as a match in the query result. Only members that score higher than or equal to the value of this parameter are returned. To return all candidates, use a large negative value, for example -999.
Patient Demographics Supplier
Pattern Parameter Default Description
Supplier Application This parameter defines the Sending Application value that is entered in the MSH segment of acknowledgments and responses that are sent from the Patient Demographics Supplier.
Supplier Facility This parameter defines the Sending Facility value that is entered in the MSH segment of acknowledgments and responses that are sent from the Patient Demographics Supplier.
Patient Demographics Consumer
Pattern Parameter Default Description
Application This parameter defines the Receiving Application value that is entered in the MSH segment of acknowledgments and responses that are sent from the Patient Demographics Supplier.
Facility This parameter defines the Receiving Facility value that is entered in the MSH segment of acknowledgments and responses that are sent from the Patient Demographics Supplier.

All the pattern parameters are listed with their default values. Where no default is indicated, the default value is an empty string.
